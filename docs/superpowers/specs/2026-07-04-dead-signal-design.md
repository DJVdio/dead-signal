# Dead Signal（死信号）— 游戏设计文档

日期：2026-07-04
状态：已与设计者确认的 v1 设计

## 1. 定位

- 中短流程**剧情向**生存经营游戏，对标《这是我的战争》（This War of Mine）。
- 流程 10~15 小时，以一周目体验为核心，不追求随机重玩性。
- 俯视角 2D，像素风格；个人项目，暂不考虑发行，内容尺度不设平台约束（重度题材保留，采用"暗示+事后现场"的呈现手法）。

## 2. 技术选型

- **引擎**：Godot 4.x + C#。
- **架构原则**：数据驱动。武器、护甲、部位、角色、信件、事件、关卡遭遇全部定义为数据文件（Godot Resource 或 JSON），代码只实现规则引擎。加内容不改代码。
- **美术**：像素风（16/32px），原型期使用现成素材包（itch.io / Kenney 等），后期统一重制。角色装备外观采用 paper doll 叠层。
- **数值策略**：设计期不死磕数值。原型可跑后，编写命令行蒙特卡洛模拟器，把武器×护甲×多层结算的期望伤害拉表微调。

## 3. 世界观与主线

某区域爆发丧尸灾变。丧尸**白天休眠、夜晚游荡**。玩家经营一处庇护所，招揽幸存者，探索、生产、防御，等待救援。

### 主线：坏掉的电台

- 营地有一台电台，能收到军方的"救援广播"：联系上就来救援。
- 蒂诺发现电台只能收、不能发。需要一路探索，最终远征危险的**广播台**取得维修材料。
- 全程持续有新幸存者加入、有人死去。

### 双结局（含隐藏第三结局）

1. **呼叫军方（背叛结局）**：军方到来后屠杀营地、抢走全部资源，只留一名重伤者逃生等死，以其视角走完收尾剧情（人选按剧情权重/养成深度挑选，冲击最大化）。
2. **不呼叫（坚守结局）**：迎来超大规模丧尸潮+气候剧变（寒潮——终局回收温度装置，保暖装备成为救命物资）。扛下来后进入 CG，向玩家发问：这样做是否正确？配合逐个死去与活着的名字收束全篇。
3. **全灭（隐藏结局）**：终局防御战可以失败，全灭即第三结局。

### 伏笔设计（必做，否则结局 1 是廉价反转）

- 广播措辞逐渐可疑：要求上报人数、物资储量、武装情况（踩点，不是救援）。
- 探索中可发现其他呼叫过救援的营地遗址：弹壳遍地、物资搬空、血字"别信电台"。
- 蒂诺（修电台的人）支线中持续表达怀疑。
- 目标：让"呼叫与否"成为有重量的真选择，选择呼叫的玩家是"无视了所有警告的人"。

## 4. 核心循环

### 白天（丧尸休眠）

- 玩家从**世界地图**选点，派出 **≤3 人小队**进入**手工设计的独立关卡**（医院、超市、金手指帮据点、广播台等）探索：搜刮物资、遭遇剧情、招募幸存者。
- 地点远近决定往返耗时，**天黑前必须回营**。
- **人类劫掠者只在白天活动**（丧尸睡了人才敢出门）：主力外出时营地可能遇袭。"带谁出门、留谁守家"是每日核心权衡。

### 夜晚（丧尸游荡）

- **每晚实时度过**，时间加速档位 1x/3x/8x，事件发生自动拉回 1x。
- 玩家安排：站岗、生产、修理、读书/训练、角色对话事件、睡眠。
- 丧尸袭营时进入实时+战术暂停的**防御战**。
- 幸存者作息为**轮班制**：探索队夜里睡、留守队白天睡。人手永远紧缺是常态压力源。

## 5. 战斗系统

### 时间模式与操控

- **实时 + 战术暂停**（类 RimWorld），点选指挥小队。
- 暂停中可下达移动、攻击目标、**瞄准部位**（耗时换取指定部位命中）等指令。

### 部位命中

- 命中部位按**体积加权随机**分配，瞄准指令改变权重。
- 部位粒度到细部位：左右眼、鼻、下巴、左右手/臂/腿等，用于伤残效果与战报叙事。
- **每部位独立 HP**；部位为**树形层级**（手挂上臂下、脚挂大腿下、眼/鼻/下巴挂头下）。
- 部位归零后果：**头/颈/躯干归零致死；四肢归零致残；眼归零致盲**（双眼皆失方为全盲）；鼻/下巴等仅毁容、无系统后果。
- **切除连带**：切除某部位时其所有下游部位一并失去（切上臂→左手随之失去），两处挂载装备掉落。
- （部位 HP 值与体积命中权重为原型期拟定，命令行模拟器拉表微调。）

### 武器

- 属性：伤害区间、穿透力、伤害类型（锐器/钝器）；远程另有误差角、攻速。
- 单手/双手之分；单手武器可**双持**（如手枪+匕首）。
- **命中模型（没有"命中率"）**：近战**必中**；远程有**误差角**——向以准星为轴、半角为误差角的锥形范围内随机一个方向射击，锥角越小越准（手枪≈3°、冲锋枪≈6°、步枪≈2°、狙击≈0.5°，拟定待调）。弹道飞行与碰撞判定归实时层。
- **双持惩罚**：两手各按自身武器攻速独立出手，两手都吃惩罚——近战双持**攻速为 70%**；远程双持**攻速 70% 且误差角 +25%**；单持不惩罚。
- 锐器：伤害上下限差距大、穿透高（匕首 9%、短剑 12%、长剑 18%、重剑 24%；土制枪 10%、手枪 15%、冲锋枪 18%、步枪 21%、狙击枪 70%）。
- 钝器：伤害区间窄、穿透低（棍棒 7-9 / 3%、尖头锤 5%、重型破甲锤 20%）。
- 狙击类高穿透武器靠弹药稀缺、上弹时间、限定出现约束。

### 护甲与逐层结算

- 护甲属性：锐器防御、钝器防御、重量、所属层（贴身/外套/装甲层）。
- 锐防普遍约为钝防两倍；厚重板甲锐防可达钝防数倍。
- **负重分段**（重量惩罚，遏制无脑堆甲）：角色有负重上限（由体力属性换算）。总装备重量 0~50% 上限无惩罚；50~100% 线性惩罚攻速/移速；>100% 陡峭惩罚。曲线拟定待调。

**逐层三段判定**（每层从外到内依次结算）：

1. roll 为**连续均匀分布，允许小数**。攻方在伤害区间内 roll；防方在 `[0, 适用防御 × (1 - 攻方穿透)]` 内 roll（下限为 0，保留"板甲被匕首捅穿"的小概率——护甲不是绝对保险）。
2. 攻方 roll ≥ 防方 roll：**全额伤害**，穿入下一层。
3. 攻方 roll ≥ 防方一半：**半额伤害**；若为锐器则转为钝器且穿透归零；天然钝器每层都保留自身穿透。
4. 低于防方一半：无伤害，结算终止。
5. 伤害全程按小数计算，最终作用到部位时再取整（向上取整，最低 1 伤；中途终止为 0，不适用保底）。
6. **逐层递减**：第二层起攻方在 `[0, 上一层结算后的伤害值]` 内 roll（伤害逐层衰减）。无甲直击时武器伤害直接作用到部位。

### 伤害效果

- **降解后无任何状态效果**：锐器被护甲逐层降解转成钝伤后，**不流血、不切除、不震荡、不骨折，只造成伤害**。效果资格看**实际到达部位的伤害类型**——伤害仍以锐器抵达才有流血/切除；只有天然钝器攻击才有震荡/骨折。
- 锐器：
  - **流血**（开放伤口，持续失血）——造成伤害即按概率触发，伤害越高概率越高；切除必附带出血。流血**扣角色储血量、不扣部位 HP**（部位 HP 只由直接伤害改变）；见下"储血量与失血分级"。
  - **切除**（失去部位，如左手）——单次伤害 ≥ 部位最大 HP，或命中已 0 HP 的部位；两者都要求**穿甲后实际造成 >0 伤害**（被甲完全挡下则不切除、部位保住）。**致死部位达成切除条件 = 直接死亡**（斩首/爆头/开膛，不生成断肢）。
  - **设计意图（切除率刻意偏高）**：细部位低 HP 叠加"单击 ≥ 部位MaxHP 即切除"，使强武器打中小部位频繁致残/斩断——这是**有意为之的狠辣质感，不做削弱**（不上调部位 HP、不收紧门槛）。
- 钝器：
  - **震荡**（减攻速移速）——**无论是否造成伤害**（含被甲完全挡下）皆可触发，仅对头/躯干；概率随**初始武器 roll** 越高越高（头部高、躯干低）。
  - **骨折**（减对应部位功能：手→攻速，腿→移速）——造成伤害时按概率触发。
  - **钝器效果隔着未破的甲也能生效**（甲没破人被锤懵）。
  - **锐器穿透衰减后转成的钝伤，不触发震荡/骨折**（只有天然钝器攻击才有钝器系资格）。
- **0HP 部位的钝伤磨损（永久损伤，非状态效果、不走概率）**：对**当前 HP=0** 的部位继续造成**实质钝伤**（锐转钝的降解伤害、或天然钝器；均要求穿甲后实际伤害 >0，被甲完全挡下则不磨损），**永久降低该部位 MaxHp**；MaxHp 归 0 → 部位**永久损毁**（碾碎/砸烂，区别于"切除"）。损毁连带下游部位、致死部位损毁 = 死亡；损毁**不掉落**挂载装备（碾碎的手仍套在手套里，随部位报废）。锐器（未降解）对 0HP 部位仍走切除、不磨损。磨损量 = 本次伤害 ×系数（拟定待调）。
- **储血量与失血分级**：角色有**储血量**（上限拟定，可与体型/体力挂钩）。每处流血伤口按流速持续扣储血，**多处伤口叠加流速**（断口即使切除仍持续出血）。按储血余量百分比分级（阈值拟定待调）：**<75% 轻度**（攻速/移速小幅降低）、**<50% 中度**（debuff 加重）、**<25% 重度**（**昏迷**、丧失行动）、**归零 = 出血致死**。止血/治疗为后续接口。
- （流血/骨折概率 ≈ 系数 × 单次伤害 / 部位MaxHP；震荡概率 ≈ 系数 × 初始roll / 部位MaxHP；失血流速/阈值/储血上限均系数拟定待调。震荡时限衰减与骨折恢复为后续接口。）

## 6. 装备系统

- **12+ 独立槽位**：头部、眼镜、面部、贴身上衣、外套、装甲层、左护臂、右护臂、左手套、右手套、裤子、腿甲、鞋子；武器主手/副手。
- 部位**被切除时其挂载装备掉落**到现场（可被搜刮）。
- 角色有**负重上限**（体力属性换算），超限按分段惩罚攻速/移速（见战斗系统·负重分段）。
- 温度、辐射**不做全局系统**，做成**章节装置**：特定关卡需要对应装备（辐射区→防化服；终局寒潮→保暖装备），让低防护功能性衣物（防化服、绒毛披风、无袖背心）有登场理由。

## 7. 角色与叙事

### 规模

- 全程登场 **10~12 名手写幸存者**（含死亡/离开），同时在营 **≤8 人**。
- **全员永久死亡，无主角**；叙事视点是"营地"本身。全灭即失败（= 隐藏结局 3）。

### 每个角色包含

1. **招募剧情**：通过特定事件获得（如第一波袭击教学战中反水的克莉丝汀），可选收留/赶走/战斗处决，各有分支。
2. **行为驱动三级成长**：如蒂诺（开局跟随的书呆子）读书速度 +15%；累计读书 48h 升 2 级 +35%；3 级 +50% 且全员受读书氛围感染 +5%。
3. **支线任务（含时限失败态）**：如克莉丝汀想报复金手指帮（她从那里逃出，被侵害的过去）。玩家长期不响应→她带走物资独自离开→日后探索金手指帮据点时发现她的尸体与惨死现场（环境叙事呈现）。
4. **信件系统**：白天幸存者给玩家写信——推进支线（克莉丝汀反复请求清剿金手指帮）、提营地需求（蒂诺想要书柜/书桌/沙发，满足后 +5% 阅读速度小任务奖励）。
5. 角色死亡后，其支线与信件相应**中断或转向**。

### 解锁与生产

- 搜集的书籍可阅读解锁**蓝图**（如枪械制造手册），夜间生产制造装备与工事。

## 8. 失败与死亡

- 任何幸存者可在探索、防御战、剧情中永久死亡；重伤可致永久伤残（切除/骨折后遗症）。
- 营地全灭 = 游戏结束（隐藏结局）。

## 9. 开发优先级（支柱排序）

1. **战斗结算引擎**（武器/护甲/部位/逐层判定）+ 蒙特卡洛数值模拟器 —— 差异化核心，风险最集中。
2. **昼夜循环骨架**：世界地图选点 → 关卡探索 → 回营 → 夜晚实时+排班+防御战。
3. **角色与信件**：先做 2~3 个角色（蒂诺、克莉丝汀）打通"招募→成长→支线→失败态"全链路。
4. 生产/蓝图、章节装置（温度/辐射）、人类袭击 AI、主线结局 —— 依次后置。

## 10. 明确不做（YAGNI）

- 程序化生成地图/角色/任务。
- 全局温度、辐射、心情模拟系统。
- 多人、模组支持、长线沙盒平衡。
- 国内平台过审适配。
