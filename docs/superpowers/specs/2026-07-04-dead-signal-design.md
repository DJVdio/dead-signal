# Dead Signal（死信号）— 游戏设计文档

日期：2026-07-04
状态：已与设计者确认的 v1 设计

## 1. 定位

- 中短流程**剧情向**生存经营游戏，对标《这是我的战争》（This War of Mine）。
- 流程 10~15 小时，以一周目体验为核心，不追求随机重玩性。
- 俯视角 2D，像素风格；个人项目，暂不考虑发行，内容尺度不设平台约束（重度题材保留，采用"暗示+事后现场"的呈现手法）。

## 2. 技术选型

- **引擎**：Godot 4.x + C#。
- **架构原则**：数据驱动。武器、护甲、部位、角色、对话气泡、事件、关卡遭遇全部定义为数据文件（Godot Resource 或 JSON），代码只实现规则引擎。加内容不改代码。
- **美术**：像素风（16/32px），原型期使用现成素材包（itch.io / Kenney 等），后期统一重制。角色装备外观采用 paper doll 叠层。
- **数值策略**：设计期不死磕数值。原型可跑后，编写命令行蒙特卡洛模拟器，把武器×护甲×多层结算的期望伤害拉表微调。

## 3. 世界观与主线

某区域爆发丧尸灾变。丧尸**白天休眠、夜晚游荡**。玩家经营一处庇护所，招揽幸存者，探索、生产、防御，等待救援。

**2026-07-06 新增规划（待实现，代码尚未落地）—— 辐射变异生物**：太阳强辐射不仅催生了丧尸，还产生了其他变异生物，包括**变异犬**（高速、成群行动）、**辐化爬行者**（大型多足生物，攀爬能力极强）、**辐化鸟类**（空中威胁，可俯冲攻击）等。变异生物比普通丧尸更难对付，部分拥有特殊能力（如远程酸液喷射、自爆）。与丧尸不同，辐射变异生物**白天也会活动**，不受日光抑制，是白天探索的另一大威胁。

### 主线：坏掉的电台

- 营地有一台电台，能收到军方的"救援广播"：联系上就来救援。
- 诺蒂发现电台只能收、不能发。需要一路探索，最终远征危险的**广播台**取得维修材料。
- 全程持续有新幸存者加入、有人死去。

### 结局矩阵（正史·用户总纲）

电台是三条结局的总闸——"收得到、发不出"（§标题主题）下，玩家如何处置这台坏电台决定终局。三条正史结局（用户总纲原话级口径，**CG/正文属用户手写剧情，本文档只定机制语义**；下列未实装细节一律标"待用户设计"）：

1. **不回复军方电台 → 第 40 天被无限尸潮屠杀（悲壮终局·必死）**：不理会军方救援广播、放任时限流逝，则第 40 天（`DeadlineDay`，拟定待调）北方**上百万尸潮南下抵达**，营地陷入**无限围攻至全员战死**——即 §5"尸潮时限与终局围攻"的到期围攻，二者为**同一事件**（尸潮一律不可战胜，D3 定稿）。玩家能做的只是让营地更久、更有尊严地扛住。全员战死后进入 CG（向玩家发问：这样做是否正确？配合逐个死去与活着的名字收束全篇）。
2. **回复军方电台 → 第 3 天被军方派来的人屠杀（背叛终局·必死）**：向军方回复求救，则**第 3 天军方派来的人抵达、屠杀营地**（新正史，替换原"军方来救援后屠杀"设定）。军方屠杀事件的触发时序、执行方式、CG 呈现**均未实装，细节待用户设计**。
3. **电台发出设备 → 向南逃离（唯一生路·二号大地图伏笔）**：若玩家取得**电台发出设备**，可**向南边的营地求救**→请求他们**临时开放"前往峡谷的路"**→**完成他们的考验**后，带上**一小部分物资向南逃离**。这是**唯一的生路**，并**为第二个大地图做伏笔**。南方营地、求救交涉、"前往峡谷的路"、考验内容、物资携带上限、逃离流程**全部未实装，细节待用户设计**。
   - **与 D1 终局冻结的衔接（待确认）**：南逃属主线终局路径，按 D1"主线到终局抉择点冻结时限"本应冻结尸潮倒计时；但"完成考验→逃离"有流程耗时，**是否仍受第 40 天硬限约束（考验拖过 40 天会否被尸潮追上）待用户确认**，勿自裁。

- **全灭（通用失败态，非独立结局分支）**：任一防御战/探索可全灭 = 游戏结束，走 §9 失败与死亡。它不是上述三条正史之一，而是玩家未走到任何结局就团灭的兜底态。

### 派系：金手指帮与哥顿

一伙成形于灾后的劫掠者帮派，是克莉丝汀支线（见 §8"首条成形支线：克莉丝汀"）的反派与情感核心。

- **帮派文化**：信奉"男强女弱"，视女性为可占有的附属。以手指侵犯女性下体为"仪式"，后演变为以异物侵犯——此即玩家在其根据地见到的克莉丝汀尸体状态之由。此为帮派的既定文化，非个别行为。
- **日记A（根据地，帮众尸旁）**：拾于**被克莉丝汀反杀的帮众尸体**旁（该尸体各分支恒在，见 §8.7）。日记主人与其朋友只是两个普通帮众，记述灾后二人互助求生、参与对俘获女性的轮奸与折磨，以及**成员视角**的"金手指帮"命名由来。
- **帮主哥顿**：在玩家到达前**早已上吊自杀**。其**上吊尸体与日记B** 不在金手指帮根据地，而在**另一处探索目的地「守望者森林小屋」**——哥顿独自走进林子，吊死在**小屋门口的老树上**（环境叙事）。日记B 揭示：金手指帮文化的**起源**、哥顿**身世**（母亲极度强势；母亲丧尸化后当着他的面杀死了他懦弱的父亲）、以及哥顿**以暴戾与恐怖掩盖自身的懦弱**；最终他看透了自己的本性、发觉虐杀女性毫无意义，遂自杀。
- **日记为可拾取 lore 物品**：日记A / 日记B 均为现场**可拾取的 lore 物品**，带回营地后阅读。

## 4. 核心循环

### 白天（丧尸休眠）

- 玩家从**世界地图**选点，派出 **≤3 人小队**进入**手工设计的独立关卡**（医院、超市、金手指帮据点、广播台等）探索：搜刮物资、遭遇剧情、招募幸存者。
- **现有探索目的地（`WorldMapPanel`，行程为拟定待调）**：超市 / 医院 / 药店 / 住宅区 / 加油站 / 金手指帮根据地（剧情，§8.7）/ 守望者森林小屋（剧情，§8.7）/ **河边小屋（前中期，栓动猎枪，用户拍板）** / **联合收割机仓库（前中期，《进阶木匠技术》，用户拍板；《木匠入门》改神秘商人出售）** / **城市之巅瞭望观景台（前中期，剧情调查点——望远镜可眺望正北南下尸潮，触发尸潮时限知情，规则见 §5"尸潮时限与终局围攻"）**。
  - **探索点搜刮机制（MVP）**：探索关（`TestExploration`，各目的地共用场景仅换触发点）**无关卡内点击式搜刮容器**（点击搜刮是营地专属）——探索点取物走**发现点式**：小队走入一处搜刮点（枪柜 / 工具柜 / 床底箱 / 阁楼箱等）即**踏入触发**，整批掉落入库并弹一段**环境叙事**（一次性、跨关持久去重）。投放/叙事纯逻辑见 `ExplorationCache`（draft），铺设见 `TestExploration`。
- 地点远近决定往返耗时，**天黑前必须回营**。
- **人类劫掠者只在白天活动**（丧尸睡了人才敢出门）：主力外出时营地可能遇袭。"带谁出门、留谁守家"是每日核心权衡。
- **2026-07-06 新增规划（待实现，代码尚未落地）—— 白天强辐射**：白天太阳辐射极强，探索队需要**辐射抗性设备**（防化服、抗辐药剂等）才能安全外出。辐射值累积过高会导致角色受到辐射伤害（不可逆的辐射病）或变异。随着游戏天数推进，太阳辐射逐日增强（难度递增机制）。抗辐设备可通过探索搜刮或营地制作获得，部分关卡需要达到一定辐射抗性门槛才能进入。

### 夜晚（丧尸游荡）

- **每晚实时度过**，时间加速档位 1x/3x/8x，事件发生自动拉回 1x。
- 玩家安排：站岗、生产、修理、读书/训练、角色对话事件、睡眠。
- 丧尸袭营时进入实时+战术暂停的**防御战**（规则见 §5"守卫防御战"）。
- 幸存者作息为**轮班制**：探索队夜里睡、留守队白天睡。人手永远紧缺是常态压力源。

## 5. 战斗系统

### 时间模式与操控

- **实时 + 战术暂停**（类 RimWorld），点选指挥小队。
- 暂停中可下达移动、攻击目标、**瞄准部位**（耗时换取指定部位命中）等指令。
- **三阵营描边（表现层，用户拍板）**：角色脚下恒绘一圈阵营色底环，供战场敌我速读——**己方白 / 中立蓝（如神秘商人）/ 敌对红**。阵营色由现有阵营体系推导（视角=玩家营地 Survivor，敌对与否走 `Factions.IsHostile`，中立=非己方且不敌对），不新增字段、不对具体角色特判。选中态另以绿环画于阵营环之外形成双环（选中高亮压在外层、阵营色内环恒示；此并存层级为保守拟定，待确认）。

### 部位命中

- 命中部位按**体积加权随机**分配，瞄准指令改变权重。
- **两级命中判定（区域→子部位）**：部位树在根与现有部位之间新增一层"**大区域**"（手部、上臂、腿、脚、头部、躯干等）。命中流程分两级——
  1. **第一级**：按各大区域的体积权重加权选中一个大区域；
  2. **第二级**：在大区域内按子部位体积权重加权选中具体子部位。
  - **手部大区域** → 五指之一（左手拇指/食指/中指/无名指/小指，右手同理）vs **手掌**（手部本体，占大部分权重）。
  - **脚部大区域** → 五趾之一（左脚拇趾/第二趾/中趾/第四趾/小趾，右脚同理）vs **脚掌**（脚部本体，占大部分权重）。与手→五指对称。
  - **头部大区域** → 左眼、右眼、鼻、**左耳**、**右耳**、下巴、头颅。
  - 采样对外接口签名不变（`VolumeWeightedHitSelector`），内部改为两级。
- 部位粒度到细部位：左右眼、鼻、下巴、**左右耳**、左右手/臂/腿/脚、**五指**、**五趾**等，用于伤残效果与战报叙事。五指/五趾为独立 body part（低 HP、低权重）。
- **每部位独立 HP**；部位为**树形层级**（**头直接挂躯干下**、手挂上臂下、脚挂大腿下，眼/鼻/耳/下巴挂头下，五指挂手部下，五趾挂脚部下）。
- **移除"颈部"**：不再设独立颈部位（无对应装备槽防护，见 §6），头直接挂躯干下；原颈的命中权重**并入躯干**。
- 部位归零后果：**头/躯干归零致死；四肢归零致残；眼归零致盲**（双眼皆失方为全盲）；鼻/下巴等仅毁容、无系统后果。
- **切除连带**：切除某部位时其所有下游部位一并失去（切上臂→左手随之失去），两处挂载装备掉落。
- **各部位 HP 表（单击伤害 ≥ 部位 MaxHp 即切除，拟定待调）**：头颅 16 / 眼睛 6 / 鼻子 12 / 下巴 12 / 耳朵 8 / 躯干 28 / 四肢（上臂·大腿）21 / 手·脚 16 / 手指·脚趾 10。
- （部位 HP 值与体积命中权重为原型期拟定，命令行模拟器拉表微调。）

### 武器

- 属性：伤害区间、穿透力、伤害类型（锐器/钝器）；远程另有误差角、攻速。
- **持握模式（GripMode，已实现）**：武器分**单手 / 双手**两类。角色的三种持握姿态——
  - **双持两把单手武器**（如手枪 + 匕首）：**两把都须可双持**（单手类）；两手各按自身攻速独立出手，攻速 **×0.70**，远程另叠**误差角 ×1.25**（惩罚见下"双持惩罚"）。
  - **双手持一把武器攻速 +15%**：可单手可双手的武器（如短剑）双手持时吃 +15% 加成；只能双手持的武器（如消防斧）也吃此 +15%（数值拟定待调）。
  - 单手单持：无加成无惩罚。
- **命中模型（没有"命中率"）**：近战**必中**；远程有**误差角**——向以准星为轴、半角为误差角的锥形范围内随机一个方向射击，锥角越小越准（手枪≈3°、冲锋枪≈6°、步枪≈2°、狙击≈0.5°，拟定待调）。弹道飞行与碰撞判定归实时层。
- **射程与距离衰减（已实现）**：每把远程武器有 **MaxRange（最大射程）**——超出即不可开火（守卫的"有效交战距离"另受哨塔/屋顶加成，见"守卫防御战"）。射程内伤害按距离线性衰减：**FalloffStart（衰减起点）以内为满伤**，其后随距离线性下降，到 MaxRange 处收敛为 **FalloffFloor（下限系数，满伤的百分比）**。每把武器的衰减曲线各异（如手枪近而陡、步枪/狙击远而缓）。**护甲在距离衰减之后再扣**——远射时武器伤害已被削减，护甲照常逐层结算，故**远射更易被甲挡下**。近战武器不受此约束。MaxRange / FalloffStart / FalloffFloor 数值拟定待调。
- **弹道命中判定 + 友军误伤**：远程弹沿采样方向直线飞，命中**路径上第一个碰撞体**（撞墙则落空）——不追踪指定目标。**允许友军误伤**（射线经过的己方也会被打中，真实向），但设**紧贴架肩豁免**：当被命中的同阵营队友与**射手**处于**紧贴间距**（阈值内，视作把枪架在队友肩上射击）时，弹道**穿过该队友继续飞**、不造成友伤；紧贴阈值之外的队友可被误伤。敌对阵营正常命中。紧贴间距阈值（≈两者碰撞半径之和 + 余量，肩宽量级）拟定待调。
- **双持惩罚**：两手各按自身武器攻速独立出手，两手都吃惩罚——近战双持**攻速为 70%**；远程双持**攻速 70% 且误差角 +25%**；单持不惩罚。
- **远程武器贴脸近战 profile（枪托）**：当目标贴脸（边缘间隙内，阈值拟定待调）时，远程武器**强制禁用远程射击、只能近战**——此时改用其**近战数值**（枪托钝击：钝器类、伤害低，贴脸**必中、无误差角**）。作为新增规则形态记入，具体近战数值拟定待调。
  - **双持逐手判定**：双持"匕首 + 手枪"被贴脸时**逐手结算**——匕首本就是近战武器，**正常近战**；手枪走**枪托**近战数值。即"手上只有远程武器的那只手"才降级为枪托，近战手不受影响。
- 锐器：伤害上下限差距大、穿透高（匕首 9%、短剑 12%、长剑 18%、重剑 24%；土制枪 10%、手枪 15%、冲锋枪 18%、栓动猎枪 16%、步枪 21%、狙击枪 70%）。
- 钝器：伤害区间窄、穿透低（棍棒 7-9 / 3%、尖头锤 5%、重型破甲锤 35%——破甲专精，全近战最高穿透，拟定待调）。
  - **破甲锤定位（收官，数值终态不再动）**：穿透 35% / 冷却 4.5s 为终态。它是**致残/打断功能型武器**（高骨折致残 + 震荡硬打断破节奏），**正面 1v1 对决偏弱（Sim ~10% vs 长剑满甲）是设计代价，非待修平衡问题**——价值在削弱重甲目标的行动力、为队友创造窗口，而非单挑斩杀。
- 狙击类高穿透武器靠弹药稀缺、上弹时间、限定出现约束。

### 攻击冷却状态机（防走 A，已实现）

为杜绝"走 A"（贴上去瞬发一击再撤，用移动刷掉出手前摇）的操作，攻击出手走一套冷却状态机。
**枪械（及一切武器）攻击模型 = 冷却→射击→冷却→射击**：循环以冷却起头，首发也要先过满一次冷却。

- **起手先满冷却**：处于**非攻击态**的角色收到攻击指令时，**先走满一次冷却**（= 该武器的有效攻速间隔）再打出**第一下**，不允许瞬发。（引擎侧 DuelEngine 与实时侧 Godot 均已对齐此口径。）
- **同目标续攻不刷新**：已处于**攻击态**、对**同一目标**重复下令攻击，**不重置冷却**——按既有节奏连续出手，重复下令无收益。
- **切目标 / 移动后重算**：**切换攻击目标**或**发生移动**后再攻击，视为重新起手，**重算冷却**（回到"起手先满冷却"）。
- 有效攻速间隔受持握模式（双持 ×0.70 / 双手 +15%）、负重、伤残/饥饿能力惩罚等影响，见相关小节。

**连发（一次"射击"打多发）**：

- 武器有**连发数**字段（默认 1＝单发）。一次"射击"＝连续打出 N 发，**每发独立锥形误差角采样 / 命中 / 伤害 roll**（连发跟枪，逐发重新瞄准目标当前位置）。
- 连发内每弹有一个很小的**连发内间隔**（拟定 ~0.06s，拟定待调）；**冷却在整轮连发之后才起算**（冷却→N 发连发→冷却→N 发连发）。因此有效 DPS＝N 发/循环，循环时长 = 冷却 + (N-1)×连发内间隔。
- **冲锋枪 = 三连发**（用户拍板）。冷却 **1.8s**、连发内间隔 0.06s（拟定待调），一次开火打 3 发、约每 1.9s 一轮，DPS ≈ 22.7（仍是全表榜首但不碾压，约 2× 次名）；三连发是它的特色（爆发点射换更长冷却）。其余枪械默认单发。

**全局节奏（慢战斗，用户拍板）**：

- 设计意图：**玩家需同时操控多名角色作战**，攻速太快则战斗瞬间出结果、玩家管理不过来。故整体攻速刻意放慢，追求"看得清、指挥得动"的可管理节奏。
- **近战锚点（拟定待调）**：匕首冷却 **1.4s**（×2）、长剑 **3.6s**（×3.3）；近战按 ×2~3.3 随武器沉重度插值（破甲锤外推离谱故按手感封顶 4.5s，全近战最慢）。
- **枪械冷却倍率经用户回调 ×5~5.6 → ×3~4（原步枪 4.5 / 狙击 7.5 两锚作废）**：手枪 **2.0**、冲锋枪轮冷却 **1.8**、步枪 **2.8**、狙击 **5.0**、土制枪 **3.5**（拟定待调）。目标"枪械正面 DPS 压制多数近战但非碾压"——步枪 vs 长剑复验 ~80%。
- **栓动猎枪（新增，民用猎枪，与军用步枪并存，用户拍板两把都有）**：冷却锚 **4.5s**（用户最初给它的），单发郑重；伤害/穿透/射程介于步枪与土制枪之间（16-28 / 穿 16% / 射程 420，拟定待调）。**获取来源：前中期探索点「河边小屋」的枪柜搜刮点**（用户拍板"按直觉搭配投放"，猎人小屋语境）——踏入即入库，投放/环境叙事见 `ExplorationCache`（draft 拟定待调）。
- **节奏放慢适用于全场（含敌方）**：只放慢玩家武器会单方削弱玩家（Sim 复验：人 vs 丧尸 91%→57%）；敌方天生武器/袭击者须同步放缓（丧尸爪击 1.3→2.3s，复验恢复 ~90%）。消费层（Godot Pawn/Zombie/Raider）冷却已改读武器表权威间隔，慢节奏在实际游戏落地。
- 数值随 Sim 拉表校准，均**拟定待调**。

### 护甲与逐层结算

- 护甲属性：锐器防御、钝器防御、重量、所属层（贴身/外套/装甲层）。
- 锐防普遍约为钝防两倍；厚重板甲锐防可达钝防数倍。
- **按部位覆盖（已实现）**：每层护甲可**只覆盖部分具体身体部位**（用具体部位名集合表达，支持左右分，如仅左手）；**默认（不指定覆盖集合）= 全覆盖**（向后兼容旧数据）。逐层结算时，某部位被命中只应用**覆盖了该部位**的护甲层，不覆盖该部位的层直接跳过（视同该处无甲）。举例：劳保手套是**一副两件**（左手套 / 右手套），左手套只防左手（含左手五指）、右手套只防右手，命中脚/躯干时手套不参与结算。注意"覆盖哪些部位减伤"与"占用哪个装备槽"是两回事（见 §6 装备槽系统）。
- **负重分段**（重量惩罚，遏制无脑堆甲）：角色有负重上限（由体力属性换算）。总装备重量 0~50% 上限无惩罚；50~100% 线性惩罚攻速/移速；>100% 陡峭惩罚。曲线拟定待调。

**逐层三段判定**（每层从外到内依次结算）：

1. roll 为**连续均匀分布，允许小数**。攻方在伤害区间内 roll；防方在 `[0, 适用防御 × (1 - 攻方穿透)]` 内 roll（下限为 0，保留"板甲被匕首捅穿"的小概率——护甲不是绝对保险）。
2. 攻方 roll ≥ 防方 roll：**全额伤害**，穿入下一层。
3. 攻方 roll ≥ 防方一半：**半额伤害**；若为锐器则转为钝器且穿透归零；天然钝器每层都保留自身穿透。
4. 低于防方一半：无伤害，结算终止。
5. 伤害全程按小数计算，最终作用到部位时再取整（向上取整，最低 1 伤；中途终止为 0，不适用保底）。
6. **逐层递减**：第二层起攻方在 `[0, 上一层结算后的伤害值]` 内 roll（伤害逐层衰减）。无甲直击时武器伤害直接作用到部位。

### 伤害效果

- **降解后无任何状态效果**：锐器被护甲逐层降解转成钝伤后，**不流血、不切除、不震荡、不骨折，只造成伤害**。效果资格看**实际到达部位的伤害类型**——伤害仍以锐器抵达才有流血/切除；只有天然钝器攻击才有震荡/骨折。
- 锐器：
  - **流血**（开放伤口，持续失血）——造成伤害即按概率触发，伤害越高概率越高；切除必附带出血。流血**扣角色储血量、不扣部位 HP**（部位 HP 只由直接伤害改变）；见下"储血量与失血分级"。
  - **切除**（失去部位，如左手）——单次伤害 ≥ 部位最大 HP，或命中已 0 HP 的部位；两者都要求**穿甲后实际造成 >0 伤害**（被甲完全挡下则不切除、部位保住）。**致死部位达成切除条件 = 直接死亡**（斩首/爆头/开膛，不生成断肢）。
  - **设计意图（切除率刻意偏高）**：细部位低 HP 叠加"单击 ≥ 部位MaxHP 即切除"，使强武器打中小部位频繁致残/斩断——这是**有意为之的狠辣质感，不做削弱**（不上调部位 HP、不收紧门槛）。
- 钝器：
  - **震荡（硬打断，非攻速衰减）**——**无论是否造成伤害**（含被甲完全挡下）皆可触发，仅对头/躯干；概率随**初始武器 roll** 越高越高（头部高、躯干低）。命中触发后目标进入硬打断：期间**不能出手、移动速度 ×0.1（−90%）**；且**已积累的冷却进度清零**——打断结束后**从零重走一次完整冷却**（例：3s 冷却武器冷却已推进 2.9s 时吃 2.5s 震荡 → 下次出手 = 打断结束 + 3.0s 满冷却）。打断时长按是否击穿分档：**击穿/无甲命中 = 2~5 秒 roll**（均匀分布）；**未击穿护甲（被甲完全挡下，"甲没破人被锤懵"）= 固定 1 秒**（均拟定待调）。
    - **震荡抗性（防死锁）**：吃到一次震荡后，其**打断 + 首轮重走冷却**期间，再次被震荡的**触发概率 ×0.25（75% 抗性，拟定待调）**；该窗走完即恢复无抗性（抗性窗内再中震荡则重新打断、下轮冷却同样带抗性）。避免慢节奏下被连锁震荡完全锁死。震荡为战斗内临时态，不进营地伤病系统。
  - **骨折（持久致残，直到手术）**——造成伤害时按概率触发，落到部位持久态（沿用健康页签/手术链）。惩罚随治疗状态分档：**未治疗**手部骨折 → 操作能力 ×0.7（−30%，含攻速）、腿/脚骨折 → 移动速度 ×0.7（−30%）；**已治疗**（手术成功、愈合中）→ 惩罚**减半为 −15%（×0.85）**；**痊愈** → 归零。多处骨折**乘算叠加**、设能力下限（均拟定待调）；与断手/断指的残疾净惩罚相互独立叠乘。（"已治疗"状态由医疗层在骨折手术成功时回写躯体，运行时接线属 HealthConditions↔Body 集成波。）
  - **钝器效果隔着未破的甲也能生效**（甲没破人被锤懵）。
  - **锐器穿透衰减后转成的钝伤，不触发震荡/骨折**（只有天然钝器攻击才有钝器系资格）。
- **0HP 部位的钝伤磨损（永久损伤，非状态效果、不走概率）**：对**当前 HP=0** 的部位继续造成**实质钝伤**（锐转钝的降解伤害、或天然钝器；均要求穿甲后实际伤害 >0，被甲完全挡下则不磨损），**永久降低该部位 MaxHp**；MaxHp 归 0 → 部位**永久损毁**（碾碎/砸烂，区别于"切除"）。损毁连带下游部位、致死部位损毁 = 死亡；损毁**不掉落**挂载装备（碾碎的手仍套在手套里，随部位报废）。锐器（未降解）对 0HP 部位仍走切除、不磨损。磨损量 = 本次伤害 ×系数（拟定待调）。
- **储血量与失血分级**：角色有**储血量**（上限拟定，可与体型/体力挂钩）。每处流血伤口按流速持续扣储血，**多处伤口叠加流速**（断口即使切除仍持续出血）；战斗内实时失血流速已下调一档（每伤口/秒 0.8→0.55 draft，慢节奏下放宽多伤口抢救窗，关键对决胜率漂移 <1pt）。按储血余量百分比分级（阈值拟定待调）：**<75% 轻度**（攻速/移速小幅降低）、**<50% 中度**（debuff 加重）、**<25% 重度**（**昏迷**、丧失行动）、**归零 = 出血致死**。止血 / 治疗见下"医疗与手术系统"。
- （流血/骨折概率 ≈ 系数 × 单次伤害 / 部位MaxHP；震荡概率 ≈ 系数 × 初始roll / 部位MaxHP（抗性期再 ×0.25）；失血流速/阈值/储血上限、震荡时长 2~5s / 抗性 0.25 / 移速 0.1、骨折能力 0.7 均系数拟定待调。震荡打断+冷却清零+抗性、手/腿骨折能力惩罚已落地；骨折恢复走手术链。）
- **2026-07-06 新增规划（待实现，代码尚未落地）—— 辐射伤害**：拟新增辐射伤害类型。效果为**持续伤害（DoT）**，类似流血但**不可通过止血手段移除**（辐伤只能靠抗辐药缓解/清除）。辐射伤害直接作用于部位 HP，不触发流血、骨折、震荡等效果。辐射抗性作为独立的护甲属性（不计入锐器/钝器防御），防化服等抗辐装备单独结算辐射伤害减免，遵循 §5 逐层判定规则。

### 伤残能力与假肢系统

切除/伤残不再只是叙事，而是驱动两条**能力**数值：

- **操作能力**：影响攻速（武器出手间隔）、生产速度、开锁/修理等精细操作的成功率。
- **移动能力**：影响移速、闪避/走位效率。

两只完好手 = 满操作能力（100%），单手 = 50%；两条完好腿 = 满移动能力（100%），单腿 = 50%。

**切除惩罚表**：

| 切除部位 | 惩罚 | 说明 |
|----------|------|------|
| 一只手（左/右） | **操作能力 -50%** | 精准操作/双手任务受限 |
| 一条腿（左/右） | **移动能力 -50%** | 移动速度减半；切腿连带该侧脚一并失去 |
| 一只脚（左/右，脚本体） | **移动能力 -50%** | 该侧移动单元丧失，与断腿同惩罚（覆盖脚趾累计） |
| 一根脚趾 | **移动能力 -2%**（该脚累计） | 按脚独立累加 |
| 两只手 | **操作能力 -100%** | 无法执行需双手/精细操作的任务 |
| 两条腿 | **移动能力 -100%** | 无法移动（需被抬/轮椅） |
| 一根手指 | **操作能力 -7%**（该手累计） | 按手独立累加 |
| 一条上臂（连带手） | **操作能力 -50%**（按手计）+ 手部连带失去 | 与切手同惩罚 |

- **手指与手的关系**：手指为独立子部位（见"部位命中"两级判定）。同一只手 5 指全断累计 **-35%**；一旦整只**手掌/手部本体失去则扣 -50%**，且**覆盖**手指累计（一只手操作惩罚上限即 -50%）。手部失去时其下五指连带消失，**不再另计手指惩罚**。
- **腿、脚与脚趾的关系**：脚是**独立部位**，并下探二级判定到 **5 根脚趾**（与手→五指对称）。移动能力左右两侧各占 50%：
  - 该侧**整条腿失去 或 整只脚（脚本体）失去** → 该侧 **-50%**。腿在脚之上，**切腿连带脚一并失去**（"腿没了脚自然也会没"）。
  - **脚趾**：每根 **-2%** 移动（该脚累计）。一只脚 5 趾全断而**脚本体仍在** = **-10%**，该侧仍保有 40% 移动。
  - 每侧**上限 -50%**：脚本体失去即 -50%，**覆盖并不再叠加**该脚脚趾累计。

**假肢系统**：

假肢的恢复百分比是**相对于单肢能力**的比例（一只手 = 全局操作能力的 50%），而非全局能力的百分比。

- 恢复值 = 假肢等级百分比 × 50%（单肢能力）；净惩罚 = -50%（单肢失去）+ 恢复值。上限锁定 **100%**（不超额恢复）。

| 假肢等级 | 恢复（占单肢）→（占全局） | 断手/断腿后净惩罚 | 获取方式 |
|----------|--------------------------|------------------|----------|
| **木制假肢**（木手/木腿） | 25% × 50% = 12.5% | -50% + 12.5% = **-37.5%** | 营地制作（木头+工具） |
| **简易假肢** | 50% × 50% = 25% | -50% + 25% = **-25%** | 搜刮/交易/中级制作 |
| **仿生假肢** | 75% × 50% = 37.5% | -50% + 37.5% = **-12.5%** | 高级搜刮/广播台/稀有战利品 |

- 假肢装备在被切除部位对应的空槽位上（取代该部位）。
- 装上假肢后**不能再次被切除**；假肢**无 HP、不会损坏**（暂定）。
- **无假肢**状态下角色以残疾状态活动，操作/移动能力直接按切除惩罚扣减。
- 双手/双腿假肢同理叠加，上限锁 100%。
- （能力惩罚系数与假肢恢复比例为拟定值，后续用 Sim 校准。）

### 医疗与手术系统

伤病**不自愈**，需主动医疗；治疗是营地经营对战斗代价的另一条回应（假肢之外）。

- **伤病随昼夜恶化（不自愈）**：**流血 / 骨折不会自行痊愈**，随昼夜相位持续恶化——
  - **未处理出血按伤口大小分三层梯度**（数值**拟定待调**；引擎不逐伤口记武器类型，以**部位尺寸 + 初始严重度**为可判定代理）：
    - **① 致命大伤**（命中躯干 / 头 / 上臂 / 大腿等大部位）→ 持续失血直至**失血致死**（储血分级见上"储血量与失血分级"）。**仍必死、狠度不降**，只是恶化速度放缓、死亡时间线**从约第 5 昼夜后移到约第 7 昼夜**（放宽玩家操作窗）。即便该肢因感染坏疽被截肢，残端仍持续失血 → 未处置致命伤终究失血致死（感染不得靠截肢救活）。
    - **② 中等非致命小伤**（手 / 脚等一般咬伤 / 锐器伤）→ 出血严重度**封顶在阈值之下、不会失血致死**，但**不会自愈、必须手术闭口**；愈合前始终是开放感染源，放任可能溃烂感染 → 坏疽截肢。
    - **③ 很小的伤（可自愈）——仅微小伤口**（指 / 趾 / 眼 / 面 / 耳等**微小部位**，或严重度低于阈值的**擦伤级**）→ 不会失血致死，且**新鲜期结束后自行闭合、无需手术**（Body 侧走 StopBleed 止血同步）。是有限概率赌局：**赌赢**（新鲜期内没感染）= 伤口自愈、零成本；**赌输**（新鲜期内中招）= 伤口仍自愈闭合，但感染作为独立病状继续（须抗生素，否则坏疽截肢）。**只有微小伤口享受自愈，②中等小伤与①大伤都不自愈。**
  - **开放伤口** → **感染** → 恶化为**坏疽（肢体截肢致残）或败血症（非肢体致死）**。感染是**有限概率赌局**而非必然（数值拟定待调）：
    - **几率按伤口大小缩放**——**越小的伤越不容易感染**（大部位系数高、手 / 脚 中、指 / 趾 / 面低），乘出血严重度。让玩家在抗生素有限时**赌"省不省这一瓶"**（放任小伤坏疽率落在"值得赌但会翻车"区间：手部咬伤约三成、指部约两成，拟定待调）。
    - **只在伤口"新鲜期"内可感染**——伤口存在的**头几昼夜**才有感染风险，之后身体壁垒化、不再新感染。故**放任小伤不会累积到必然坏疽**（旧模型逐日累积→100% 坏疽已废）。感染一旦发生，时间线较短（未治约第 4 昼夜封顶）。
  - **止血 ≠ 无菌**：已手术止血的伤口在**新鲜期 / 愈合闭口前仍保留感染窗**（几率折减）——术后仍需卧床照护 / 抗生素，否则可能坏疽截肢。
  - **骨折** → 拖延不治 → **畸形愈合致残**。
  - **两道独立关卡**：**手术**根治伤口（止血 / 接骨）、**抗生素**压制感染（消毒），互不替代——手术止住血但不消毒仍会烂，抗生素压住感染但伤口不手术永不愈合。感染 / 疾病用**药品**治疗（去技能，**固定疗效系数**，不看施术者能力）；卧床休养减缓感染恶化但非根治。

- **手术点数机制（医疗点池 P）**：处置流血 / 骨折走一次手术，先算可用医疗点池 P：

  **P =（基础 15 + 床位 10 + 手术材料 + 施术者医疗书加点）× 操作能力 ×（自体手术 0.60）**

  - **基础 15**：**所有角色都能做手术，不看技能**——人人自带基础 15 点。
  - **床位 10**：在床上进行 **+10**。
  - **手术材料**（按伤情，可叠）：
    - **流血**：绷带 **15** / 针线 **15**（两者可叠加），或**急救包 60**（独占，不与前者叠）。
    - **骨折**：夹板 **25**，或**急救包 60**（独占）。
  - **施术者医疗书加点**：施术者**读过《野外生存指南》** **+5**。
  - **操作能力**：乘算施术者当前操作能力（残疾 / 饥饿惩罚照常拉低）。
  - **自体手术 0.60**：给自己动手术时整体 **×0.60**。

- **准入与判定**：
  - **P < 15 不允许手术**——提示"现状不支持进行这场手术"。
  - **分段 roll 出效率**（roll 值 = 效率% = 术后恢复效率%）：
    - **P ≤ 100** → 在 **[0, P]** 内 roll；
    - **100 < P ≤ 200** → 在 **[P−100, 100]** 内 roll；
    - **P > 200** → 在 **[100, P−100]** 内 roll（效率可 > 100%）。
  - **roll ≤ 10 = 手术失败**（材料照常消耗）。
  - roll 出的**恢复效率**驱动术后愈合速度 / 恢复程度。**骨折愈合已提速**（拟定待调）：齐装 + 卧床骨折康复约 **7 昼夜**（原偏拖的 ~13 昼夜下调），减轻中短流程的人力节奏冲击；出血愈合本就较快，未动。
  - **手术成功即刻恢复 5%**（拟定待调）：任何成功手术（**含擦边低效率成功**）当场对该伤 severity 立减 5 个百分点，给成功一个即时回报、缓和擦边成功康复过久。
  - **隔日可重做手术**（拟定待调）：伤病距上次手术**超过一天**即可**再次手术**——重新 roll 恢复效率并**覆盖**旧值（**双向风险**：旧效率高时重做可能刷成更低，故"运气好就别重做"）；材料照常消耗，重做同样吃即刻 5%。冷却内不可重做。**重做失败保守**：仅白费材料，不破坏已有愈合效率 / 闭口进度（区别于初次"失败=未止血"，因伤口已闭合）。用途：擦边成功的长尾康复可靠重做去赌一个更好的效率，代价是多耗医疗耗材（资源博弈）。
  - **睡床 +10pp 加算恢复速度**（拟定待调）：术后愈合当昼夜**在床上睡觉休息**（而非地铺）→ 恢复效率**加算 +10 个百分点**（加算非乘算，如 33%→43%）再折愈合速度，与即刻 5% 独立叠加。（注：地铺 vs 床的营地建模尚未落地，当前实现暂将卧床休养一律视作睡床，**待床位建模后按床/地铺区分**——待确认。）

- **不对玩家展示数值**：医疗点 P、roll 值、恢复效率等**均不对玩家显示**，UI 只给**模糊描述**（如"处置得当 / 勉强稳住 / 情况不妙"；已手术卡显示"恢复良好 / 平稳 / 缓慢"+ 距上次手术昼夜，供判断是否重做）。
- （基础 / 床位 / 各材料点数、医疗书加点、自体系数、失败阈值、分段区间**均为"拟定待调"**，后续以 Sim / 实机校准。）

### 昼夜相位状态机

一天由 **8 个相位**顺序流转、循环推进（`GameClock` / `DayPhase`）。**DawnMeal / DuskMeal 两个聚餐相位已落地**（一次昼→夜、一次夜→昼，一天两次聚餐）：

**DayPrep → DayTravel → DayExplore → DayReturn → 【DuskMeal】 → NightPrep → NightAct → 【DawnMeal】 →（回到）DayPrep**

- **Day 计数**：每次进入 **DayPrep** 时 `Day += 1`（起始 Day=0，故首个 DayPrep 显示"第 1 天"）。NightAct 结束经 DawnMeal 后绕回 DayPrep，进入下一天。
- **各相位时间流速（`Engine.TimeScale`）**：
  - **DayPrep / DayReturn / NightPrep / DawnMeal / DuskMeal** —— 强制暂停（TimeScale=0）。这些是模态面板/过渡窗口，玩家的暂停键与 1x/3x/8x 加速档在此期间不可切换。
  - **DayTravel** —— 固定 **8x** 快进（表现往返路程耗时，不受玩家档位影响）。
  - **DayExplore / NightAct** —— 采用玩家选择的 **1x/3x/8x** 档位，可暂停。
- **各相位职责与流转条件**：
  - **DayPrep**：新的一天开始，唤醒全员，弹出探索队编队面板；玩家确认后手动进入 DayTravel。
  - **DayTravel**：固定 8x 快进，历时 `TravelTimeSeconds`（默认 120s），到时自动 → DayExplore。
  - **DayExplore**：加载探索关卡，历时 `DayLengthSeconds`；剩余时间 ≤ `WarningBufferSeconds`（默认 300s）时触发一次"天黑前回营"警告；时间耗尽自动 → DayReturn。
  - **DayReturn**：卸载探索关卡，进入睡眠过渡（见下），过渡完成后 → DuskMeal。
  - **DuskMeal（聚餐）**：探索队返回后，**全员强制聚集**（含刚回的探索队），走"暂停变暗 → 食物分配面板 → 吃饭动画（找座/站着吃 + 世界气泡）"聚餐流程（见下"聚餐交互流程"）。结束后 → NightPrep。
  - **NightPrep**：弹出夜间站岗编排面板；确认后再弹**读书指派面板**（把某人指派去读某本书，见 §7"读书活动"）；玩家确认后进入 NightAct。
  - **NightAct**：唤醒全员、守卫上岗驻守，夜晚实时度过 `NightLengthSeconds`；期间可能触发丧尸袭营防御战（见 §5"守卫防御战"）；时间耗尽自动 → DawnMeal。
  - **DawnMeal（聚餐）**：全员唤醒，**全员强制聚集**，走同一"暂停变暗 → 食物分配面板 → 吃饭动画"聚餐流程。结束后 → DayPrep（Day+1，进入下一天）。
- **聚餐交互流程（DawnMeal / DuskMeal 共通，已落地）**——用户拍板的四步：
  1. **暂停变暗**：进入聚餐相位即冻结世界（`Engine.TimeScale=0`，由相位机保证），弹出**食物分配面板**（模态外壳遮罩 = 变暗；强制流程，**无取消 / 无 ESC**，唯一出口是"开饭"）。
  2. **食物分配面板**：展示每个存活者的**头像 / 名字 / 当前饥饿档位** + **剩余食物总数**，玩家用头像旁的 **−/+** 微调各人份数。面板打开时按**现行自动分粮策略预填**（先保全员第一餐 → 余粮补最饿者，即 `FoodEconomy.Prefill`），玩家只做微调。**校验**：每人 **0~2 份**、总份数 **≤ 库存**（超支时"开饭"禁用并红字提示）。**账目**：一份 = 1 份食物；结算走 `FoodEconomy.ResolveManual`，与既有净零 / 补餐模型**同构**（份数≥1 = 第一餐净零、份数≥2 = 补餐净 +1，见下）。
  3. **吃饭动画（实时渲染）**：确认后解冻，**被分配到食物者**就近**认领空座走过去坐下吃**；**座位（板凳 / 木椅等，见 §7"座位家具"，桌子实体暂不建模）不够时**多出来的人**走到餐区边缘站着吃**。进食数秒（拟定待调）。
  4. **世界内头顶气泡**：吃饭期间在角色头顶冒**聊天气泡**（推进支线 / 营地需求 / 故事事件，替代原信件入口）。**坐着的人正常冒泡**；**站着的人聊天气泡触发概率 ×0.5（−50%）**——玩家**可能因此错过一些线索和支线**（漏听惩罚由概率承载；`StandingBubbleFactor=0.5`、坐着基础概率 `1.0`，均拟定待调）。
- **食物消耗与稀缺**：每人每餐消耗 **1 份食物**——**库存每昼夜（两餐）按在营人数扣减**，是核心生存压力源。库存不足以喂饱全员时聚餐仍照常发生，进入**分粮取舍**：预填**默认优先喂最饿者**，玩家可用 +/- 覆盖；未分到者**饥饿状态加深**。持续挨饿 → **虚弱（能力惩罚）→ 营养不良 → 饿死**（饥饿刻度见下）。食物见底会**倒逼探索队外出搜刮补给**。
    - **补餐回升（并入"每人最多 2 份"的 +/-）**：分粮先**保障全员第一餐**（净零维持），若仍有余粮，掉档者可**补第二份**（净 +1 回升，`clamp` 到各自上限）。预填按"先保第一餐 → 余粮补最饿掉档者"给出，玩家在分配面板直接把某人加到 2 份即补餐、减到 1 份即只吃第一餐——**每人每餐至多 2 份**（无独立"补餐开关"，语义并入加减号）。这让"掉档→靠双份口粮爬回上限"成为可行路径，而非单向棘轮。
  - **饥饿刻度（v2·数值化 0–6，越大越饱）**：饥饿是一条数值刻度，`0` 饿死 / `1` 营养不良 / `2` 极度饥饿 / `3` 饥饿 / `4` 有点饿 / `5` 正常 / `6` 吃撑。
    - **上限**：普通角色上限 **5（正常）**；仅"**大胃袋**"特质角色上限 **6（吃撑）**。特质系统本轮未建，仅留钩子占位——普通角色补餐 **clamp 到 5**，故 `6` 对普通角色**仍不可达、无 buff**；唯"**大胃袋**"（Cap=6）经补餐才够得到吃撑，届时吃撑作为**余粮缓冲垫**：吃撑后次日衰减只回落到"正常"（`6→5`），不继续下滑。
    - **衰减（净零模型 + 补餐回升）**：每次昼夜切换（黎明 / 黄昏两次聚餐各触发一次）**无条件 −1**（一天 −2）；**吃第一餐 +1**（净零维持）；按当次净变化一次性结算——吃满两餐净零维持（`+1−1=0`），缺餐才实际下滑。**在同一相位内再吃第二餐（补餐）= 额外 +1 回升**（`clamp` 到该角色上限 Cap），即"掉档者本相位净 +1 往回爬"。**饿死（0）是终态，补餐也不复活。**
    - **死亡**：饥饿刻度归 **0 = 饿死**（死亡，走**非战斗死亡**路径）。
    - **效果（越饿越重，数值拟定待调）**：
      - **能力惩罚（操作 / 移动，比照残疾）**：与残疾惩罚**相乘合并**（`(1−残疾)×(1−饥饿)`）——拟定阶梯：饥饿 **−10%** / 极度饥饿 **−25%** / 营养不良 **−45%**。
    - 上述能力惩罚阶梯与衰减量**均为"拟定待调"**，后续以 Sim / 实机校准。
  - **气泡交流**：见上"聚餐交互流程"第 4 步——吃饭动画期间以**世界内头顶气泡**展示（不再是聚餐面板里的文本行），**替代原信件系统作为支线入口**；条件驱动选择器按相位 / flags / 在营存活者真实状态 / 食物过滤 + 加权抽取（`meal_bubbles.json` 数据驱动，具名说话人须在餐桌前才发声）。**坐着正常冒泡、站着 ×0.5** 承载"漏听线索"的惩罚。无需邮件 UI。

### 留守者与作息睡眠（已实现）

轮班制作息（§4 的"探索队夜里睡、留守队白天睡"）的具体落地规则（`PawnRoleManager` / `CampMain`）：

- **白天（DayTravel / DayExplore）**：入选探索队的成员为 Expedition 角色随队外出；**营地全部留守者（白天非探险队成员）进入 Sleeping**——即留守队白天睡。
- **先聚餐后睡（已拍板）**：探险队回营后**不在 DayReturn 睡下**——DayReturn 仅卸载探索关卡并把归队者带回营地，随即流转 DuskMeal。全员（含刚返回的探索队与被唤起的留守者）先聚餐+气泡交流，**聚餐结束后才进入夜间睡眠**。即时序为 `DayReturn（回营）→ DuskMeal（全员用餐）→ NightPrep（站岗编排）→ NightAct`；睡眠过渡后移到聚餐之后（探险队于夜间入睡、留守者按轮班就寝）。
- **NightAct 醒来**：夜晚开始时唤醒全员（含白天留守者），按玩家的站岗分配让对应角色上岗为 Guard。

### 守卫防御战（袭营）

夜晚（NightAct）丧尸可能袭营，营地进入**实时 + 战术暂停的防御战**（见 §4 夜晚）。守卫按 NightPrep 的站岗编排上岗、驻守防御工事，与破营涌入的丧尸交战；人员伤亡由实时战斗自然产生（可致永久伤残/死亡，见 §9）。

- **营地地图（已实现）**：营地为一片**大平地**，**四面围栏围合**；**正南、正北各一扇大门**（去掉早期"三面环山"设定）。围栏与大门构成营地的实体防线。
- **围栏与大门（可破坏，HP 拟定待调）**：围栏/大门为**可破坏屏障**，各有独立 HP——
  - **基础 HP**：围栏 **150** / 大门 **250**。
  - **升级档（数值先存，升级机制后续实现）**：围栏——支柱加固 **250** · 铁皮围栏 **400** · 全金属围栏 **750**；大门——铁皮加固 **400** · 金属浇筑 **800**。
  - **大门默认关闭**（关闭时为**实心屏障**、挡路挡视）。**袭营敌人无路可走时主动攻击围栏 / 大门**，砸毁出一个**开口**后成波涌入。
- **防御工事（建造岗位）**：夜间可建造三种守卫岗位，**本轮免费建造、暂不做材料成本**——
  - **哨塔**：实心结构（挡路、需在导航网格上挖洞），为驻守**远程守卫**加成——**射程 +10%（仅远程）**、**视野 +10%**，另有**围栏 25% 抵挡**（25% 几率来袭远程攻击打在围栏上、对守卫**完全无效**）。
  - **屋顶平台**：实心结构，为驻守守卫**射程 +10%**、**视野 +10%**，且**视野计入有效交战距离**——高点远眺真正延长开火距离。
  - **暗哨**：非碰撞的驻守标记点（**不挡路**），赋予**首发**：隐蔽先手一击，仅当目标进入武器射程内才触发。
- **驻守机制**：NightAct 中被指派的守卫移动到岗位坐标驻守、就近接敌。守卫是**"驻守炮塔"式**——不脱岗远程追击，目标一旦脱离其有效交战距离即清除该目标（不追出去）。
- **袭营触发**：绑定叙事事件（**具体事件脚本后续设计**）；本轮提供公共触发钩子 + 调试触发入口。**波次规模随天数 / 营地规模递增**。丧尸从营地大门 / 围栏被砸出的开口外生成，成波涌入营地。
- **防御战结算（实时打完算后果）**：
  - **守住**：涌入丧尸全灭。
  - **损失**：丧尸破防线入营，**或**守卫全倒——**扣食物**，且守卫在实时战斗中的**伤亡照常结算**（可致永久伤残 / 死亡，见 §9）。士气非唯一后果——士气系统已移除。
- 哨塔/屋顶/暗哨加成幅度、波次规模曲线、破防线判定、结算扣减量**均为拟定待调**，后续用 Sim 校准。

### 尸潮时限与终局围攻（新增·数值拟定待调）

为防止玩家无限拖延、不推进主线（§3"坏掉的电台"），游戏设一条**世界级时限**：正北方有黑压压的**上百万尸潮正持续向南移动**，到期抵达营地即无限围攻至全灭。此为**主题增强**——"dead signal（信号发不出去）"与"末日正逼近"叠成同一种绝望；望远镜所见的南下尸潮即 §3 结局(1)"不回复军方电台→第 40 天尸潮屠杀"的同一事件——**尸潮一律不可战胜**（关系见下"与结局(1)的衔接"）。

- **隐性计时（不依赖发现）**：时限从游戏**第 1 天起隐性推进**，拟定 **DeadlineDay=40**（数值待 Sim/用户调）。**计时不依赖玩家是否知情**——未上瞭望台时限照走，"发现"只解锁**知情/显示**，不改变到期时刻（否则"不去看"即可逃避时限）。Day 计数复用 §5"昼夜相位状态机"（进 DayPrep 时 Day+=1）。
- **城市之巅瞭望观景台（前中期调查点）**：世界地图新增的剧情探索目的地（见 §4 探索点列表）。小队抵达后**用望远镜眺望正北**→触发演出（正北黑压压上百万尸潮向南移动）→置旗标 **HordeSighted**（`StoryFlags`）→**解锁 HUD 倒计时显示**。演出为一次性、跨关持久（复用探索点发现式语义，§4）。
- **倒计时显示规则**：仅在 `HordeSighted` 为真后，HUD 展示剩余天数（`DaysRemaining = DeadlineDay − day`）。**未知情前不显示倒计时、且不给任何预警**——瞭望台是唯一的知情入口。**从未上瞭望台的玩家将毫无征兆地迎来灭亡**（D2 定稿：**不做**任何不依赖瞭望台的软预警，这是有意为之的彻底黑暗向设定，非体验缺陷，勿回补预警层）。
- **到期·无限围攻终局**：`day ≥ DeadlineDay` → 尸潮抵达：营地进入**无限波次围攻**，复用 §5"守卫防御战"的 RaidWave/BreachController 语义（丧尸从大门/围栏开口成波涌入），但**波次不停轮、规模不封顶**，直至 `GameOverCondition` 判定全灭、游戏结束。**无生还路线**（有意为之的黑暗设定，非平衡问题，勿"修复"）；终局导向 §9 隐藏结局(3)全灭。
- **终局态冻结时限（D1 定稿）**：**时限只惩罚"不推进主线的拖延"**——玩家把主线推进到**终局抉择点**（电台修好、可呼叫军方/可发起坚守战）后，**HordeTimeline 冻结、被结局流程接管**：此后不再按 `day ≥ DeadlineDay` 触发到期围攻，改由玩家选定的结局（§3 结局矩阵：回复电台的背叛终局 / 不回复的悲壮终局 / 电台发出设备的南逃生路）承接。故 core-timer 需留**冻结钩子**（如终局旗标 → 停走计时/停触发到期围攻）。已推进到终局的玩家不会被时限反噬。（注：南逃线是否仍受第 40 天硬限约束**待确认**，见 §3 结局 3 衔接注。）
- **纯逻辑 API（`godot/scripts/HordeTimeline.cs`，零 Godot 依赖，Link 进 `DeadSignal.Combat.Tests`）**：
  - `enum HordePhase { Hidden, Sighted, Arrived }`
  - `static HordePhase Evaluate(int day, bool sighted)`：`day ≥ DeadlineDay` → `Arrived`；否则 `sighted` → `Sighted`；否则 `Hidden`。
  - `static int DaysRemaining(int day)`：`DeadlineDay − day`。
  - 旗标名统一 **HordeSighted**（定义落 `StoryFlags`，其余系统只消费）。
- **与结局(1)的衔接（D3 定稿）**：§3 结局(1)"不回复军方电台→尸潮屠杀"与本节到期尸潮**是同一事件**（不是两套结算）——放任时限流逝到第 40 天，即触发本节的无限围攻至全员战死。**尸潮一律不可战胜、不留可胜路径**。§3 结局矩阵已按用户总纲把它定为正史结局之一（悲壮终局·必死）。
- **主线关键物资定点/保底（D4 定稿）**：广播台维修材料等**主线必需物资**采用**定点或保底投放**（区别于普通搜刮物资的随机性），确保"必死"只惩罚**拖延**、不惩罚**运气**——玩家不会因搜刮运气差、40 天到点仍修不好电台而非战之罪暴死。**普通物资仍走随机搜刮**。本批仅定口径，实装随主线物资系统落地。
- DeadlineDay、瞭望台接触时点、波次规模曲线**均为"拟定待调"**，后续以 Sim/实机校准。

## 6. 装备系统

- **装备槽系统（设计已拍板，待实现）—— 11 个装备槽**：**头部 · 眼镜 · 面部 · 贴身层 · 外套层 · 装甲层 ·（躯干三层：贴身/外套/装甲）· 左手 · 右手 · 裤子 · 左脚 · 右脚**（武器主手/副手为独立的持握槽，不计入这 11 个穿戴槽）。这套 11 槽是上文"12+ 独立槽位"规划的**细化落定**：
  - **四肢按左右分独立槽**：因断肢机制——断某肢则该侧槽失效（无处可穿），可改装**假肢**（见 §5 假肢系统）恢复。
  - **一件装备可占多个槽**：如防毒面具 = 眼镜 + 面部；一体式板甲 = 装甲层 + 裤子。
  - **一件装备可只占一个槽**：如粗布外套 = 外套层。
  - **成对装备占对应左右两槽各一件**：如劳保手套 = 左手 / 右手两槽各一（每只只防对应那只手，见 §5 护甲按部位覆盖）。
  - **"占哪些槽"（穿在哪）与"覆盖哪些身体部位减伤"（防哪些部位）是两回事**——前者决定装备位是否可穿/是否冲突，后者决定命中某部位时哪些甲层参与结算（见 §5 护甲按部位覆盖）。
- 部位**被切除时其挂载装备掉落**到现场（可被搜刮）。
- 角色有**负重上限**（体力属性换算），超限按分段惩罚攻速/移速（见战斗系统·负重分段）。
- **温度**仍为**章节装置**：终局寒潮需要保暖装备，让功能性衣物（绒毛披风、无袖背心）有登场理由。
- **辐射（待实现规划）**：2026-07-06 新增方向，拟从原先的"章节装置"升级为**全局白天系统**（见 §4/§5）——注意这只是**规划口径，代码尚未落地**，防化服等抗辐装备与辐射抗性护甲属性均待实现。

## 7. 制作与配方系统

营地生产的完整体系：**工作台 + 工具插槽 + 材料经济 + 配方 + 武器改装**。规则引擎本轮已落地；数值全部**拟定待调**，材料 / 工具对制作品质与速度的加成、改装的材料工具成本、建造升级机制均为**后续**。本节把 §8 角色·"解锁与生产"里"书籍解锁蓝图"的一句话，细化为下述完整配方系统。**注：角色不再有通用技能——配方门槛已从"技能等级"改为"读过对应的书"，详见下"配方"与"角色能力来源"。**

### 工作台与三工具插槽（已实现）

- **营地共享一台工作台**：生产的载体，全营公用一台。
- 工作台上有**三个工具插槽——卡尺 / 锯片 / 烧杯**。工具是**搜刮到的物品**，装进对应插槽才**解锁那一类配方**：
  - **卡尺**槽 → 解锁**卡尺类**配方（精工/弓弩等）。
  - **锯片**槽 → 解锁**锯片类**配方（木工/切割等）。
  - **烧杯**槽 → 解锁**烧杯化学类**配方（火药/药水等）。
- 未装对应工具时该类配方不可制作；工具为可搜刮/可缺失的物品（丢失即该类停产）。

### 材料经济（已实现）

- 配方**消耗材料**才能产出。材料是**可搜刮入库的物品**，与武器/护甲同走物品库存。
- **材料目录（拟定）**：木料 / 破布 / 布料 / 废金属 / 金属锭 / 钉子 / 铁丝 / 生皮 / 皮革 / 火药 / 鞣制药水 / 燃料 / 骨头 / 石料 / 绳子 / 机械零件 等（目录可扩，具体配比拟定待调）。

### 配方（已实现）

- 每个配方 = **材料成本 + 需要的工具槽 + 制作者解锁条件 + 产物**。
- **解锁按制作者本人判定**（非全营共享解锁）：**读过对应的书**即为本人解锁该配方（该制作者**个人已读**）。**不再有"制作者技能门槛"——角色没有通用技能。** 对应关系：骨刀 ←《野外生存指南》、粗布背心 ←《裁缝手记》、火药 / 鞣制药水 ←《土法化学笔记》（另需烧杯工具槽）、**木椅 / 自制弓 ←《木匠入门》**（一本管两条，同构化学书；木椅另需锯片、自制弓另需卡尺）。
- **制作产出两件事**：消耗材料 → 产出物品。（不再回喂技能经验——已无技能系统。）

### 读书活动（NightAct 指派·已落地）

读书是**夜间（NightAct）指派的一项活动**，是"读过书 → 解锁配方"（见上"配方"）与角色成长（见 §8"专属效果系统"）的落地载体。

- **指派与流程**：把某人指派去读**某一本指定书** → 他**寻路走到座位坐下**读 → **耗时读满** → 该书对他标记为**"读过"**（本人解锁其对应配方）＋期间**累计阅读时间**增长，用于推进专属效果升级（如诺蒂"书虫"，见 §8）。
- **指派时机**：在 **NightPrep** 完成守卫站岗编排后弹出的**读书指派面板**里下达（相位见 §5"昼夜相位状态机"）。
- **每本书有阅读时长 `ReadHours`（拟定待调）**：读满即"读过"；读书按 NightAct 实时推进。
- **座位与无座惩罚**：读书需坐**座位**家具；**没座位仍可读，但读书速度 −10%**。
- **读书算休养、无医疗代价**：读书归入休养活动，**不触发手术医疗点消耗/风险**（区别于 §5"医疗与手术系统"的手术），不产生医疗代价。
- **读速合成（加法，与 §8 专属效果同一公式）**：`基础读速 ×（1 + 自身加成 + 全营加成汇总）× 座位系数 × 前置系数`——座位系数：有座 **1.0** / 无座 **0.90**；自身/全营加成来自专属效果（见 §8）；前置系数见下"书籍前置链"。数值拟定待调。
- **书籍前置链（通用机制·已落地）**：一本书可声明**前置书**。读者**尚未读完前置书**时读该书**不禁止**，但**读速 ×0.2**（即耗时 5 倍；系数拟定待调）——减速只增耗时、**不改读满阈值**（仍是该书 `ReadHours`），读满照常"读过"。数据驱动：链上每本书各自声明前置即可。**首条数据：《进阶木匠技术》← 前置《木匠入门》**。
- **书目扩充（本轮新增·draft）**：
  - **《木匠入门》**（木工书，`ReadHours` 拟定 20h）：读完解锁**木椅 / 自制弓**（见上配方表）。
  - **《进阶木匠技术》**（木工进阶书，`ReadHours` 拟定 28h）：**前置《木匠入门》**（未读前置读速 ×0.2）；**读完解锁效果待用户指定**（当前为占位书，暂不挂配方产出）。
  - **获取来源（用户改单后）**：
    - **《进阶木匠技术》**：前中期探索点「联合收割机仓库」的**阁楼铁皮箱（藏深，前中期难度梯度）**——踏入即入库，投放/环境叙事见 `ExplorationCache`（draft 拟定待调）。

### 角色能力来源（无通用技能）

原"技能系统"（纺织 / 机械 / 化学 / 医疗等技能项 + 初级 / 中级 / 高级等级）曾落地，现已**连根移除**——它无实际机制后果，是误加。角色**不再有通用技能**，能力改由两条承载：

- **读过的书（解锁配方）**：读完某书即为本人解锁对应配方（见上"配方"），如《野外生存指南》← 骨刀、《裁缝手记》← 粗布背心、《土法化学笔记》← 火药 / 鞣制药水、《木匠入门》← 木椅 / 自制弓。手术加点亦看"读过《野外生存指南》"（见 §5 医疗与手术系统），不设医疗技能等级。
- **每角色 authored 专属效果（框架已落地）**：个别角色的特长以**手写剧情式的专属效果**承载（与关系 / 性格同属 authored，不做程序化数值系统），如诺蒂的读书速度加成。角色**无通用技能**；专属效果分 **1/2/3 级、每角色升级条件各不相同**（手写），系统只做"**按条件播放 / 结算 authored 效果**"的框架。完整规则见 §8"专属效果系统"。

  - **近战钝器**：铁丝强化 · 钉子强化（杆/头，互斥）/ 防滑缠手（缠手）。
  - （上列括注的"部位归属"为拟定，同部位互斥的规则形态已定，具体归属与数值待调。）

### 神秘商人与货币系统（已落地，数值全 draft 待调）

- **来访节律（用户拍板）**：**每 1~5 天**有一个**神秘商人**上门。间隔在 [1,5] 天内均匀随机（可注入随机源、可复现），首访排在开局 1~5 天后（不在开局当天）。调度纯逻辑 `MerchantSchedule`（单测覆盖区间边界/顺延/复现）。
- **到访时段（用户拍板"夜晚到访、白天离开"）**：**夜访晨离**——因游戏**白天=探险队视角**（营地无操作）、**夜晚=营地视角**（玩家可右键调度角色），商人须与营地可操作窗口重合。当前实现：**NightAct**（夜晚开始、营地视角起点，与守卫上岗/读书就座同相位）派商人从地图边缘（南门外）夜里进场、走到营地中心附近约定停留点（营地照明内）停留，**交互窗口 = 整个夜晚**；**DawnMeal**（天亮、夜晚结束）商人收摊、走出画面淡出消失。**袭营夜顺延**（夜访与袭营现同为夜晚窗口，当晚有袭营则商人不来，避免"一边打劫一边摆摊"——已有袭营在进行、或第 2 夜克莉丝汀反水教学夜均顺延到次晚再试）。
- **阵营**：商人属**中立方 `Faction.Neutral`**（新增枚举值）——`Factions.IsHostile` 对中立**恒非敌对**，谁都不打它、它也不择敌；`Actor.ApplyFactionCollision` 给中立单列碰撞层（不占阵营层、只避墙）。商人**不参战、不可指派**。**保守边界（待确认）**：袭营时若商人在场则**立即离开**（不设无敌、不掉落）。三阵营描边中「中立=蓝边」由描边系统消费 `Faction.Neutral`（见 §4 三阵营描边）。
- **交互**：沿用「右键前往交互」范式——选中角色后右键点在场商人，角色寻路走到其停留点即打开**交易面板**（模态、冻结时标）。到访/离开给 `CampToast` 一行提示。
- **货币（用户拍板"和 RimWorld 一样，用白银当作货币"）**：**白银**（`Materials.CurrencyKey = "silver"`，新增 `MaterialCategory.Currency`）——照 RimWorld 直觉，交易一律以白银计价；作为一种**可堆叠的物资型货币**进现有 `Materials`/`InventoryStore` 体系（成堆持有，持币量 = 库存中该键各堆数量之和，交易走 `InventoryStore.TrySpendMaterial` 跨堆实扣）。**散落世界可搜刮**（探索点容器少量掉落 draft 方向，量级待确认）。**开局起步 40 白银（draft）**，让交易开箱可跑。
- **货架（数据驱动、可扩展，用户拍板"只卖《木匠入门》，架子可扩展"）**：`MerchantShelf` 持一批售卖条目（`MerchantOffer` = 商品 + 单价 + 库存），可 `Add` 扩展。**第一版货架 `MerchantShelf.Default()` = 《木匠入门》×1，售价 30 白银（draft）**。**卖完下架**（库存跨来访保留）；**后续来访是否补货 = 保守默认不补货（待确认）**。
- **交易结算**：`MerchantTrade.Check` 纯判定（钱够 / 钱不够含缺口 / 已售罄），`MerchantTrade.Buy` 走 `InventoryStore` **实扣白银实产商品 + 库存减一**（先判定后落地，不会半途扣钱不给货）。面板每行买入按钮：钱不够/售罄则灰显（缺口显示「差N」、售罄显示「已售罄」），价格用 §UI 语义色（买得起绿 / 买不起红）。买入后即时刷新持币量/库存/灰显。

### 已知边界与后续

- 材料 / 工具对**制作品质与速度的加成**、**改装的材料与工具成本**、**建造升级机制**（工事/工作台升级等）= 后续，本轮未做。
- **货币经济量级 = 待用户设计**：开局起步量（当前 40 draft）、《木匠入门》书价（当前 30 draft）、**各探索点容器白银掉落来源与量级**（当前未投放，仅靠开局起步量闭环）、商人停留窗口与营地自由交互时段的贴合、售罄后是否补货——均待用户拍板（见 `docs/TODO.md`）。

## 8. 角色与叙事

### 规模

- 全程登场 **10~12 名手写幸存者**（含死亡/离开），同时在营 **≤8 人**。
- **全员永久死亡，无主角**；叙事视点是"营地"本身。全灭即失败（通用失败态，见 §3 结局矩阵末/§9）。

### 每个角色包含

1. **招募剧情**：通过特定事件获得（如教学关袭营中反水的克莉丝汀，详见下文"首条成形支线：克莉丝汀"），可选收留/赶走/战斗处决，各有分支。
2. **行为驱动三级成长（= authored 专属效果，见下"专属效果系统"）**：角色无通用技能，特长以分 **1/2/3 级、每角色升级条件各异**的 authored 专属效果承载。如诺蒂（开局跟随的书虫）阅读速度 1 级 **+25%** / 2 级 **+50%** / 3 级 **+50% 且全营 +25%**（靠累计读书时长升级，阈值拟定待调）。**旧占位数值 +15 / 35 / 50% 与全员 +5% 已由此新口径覆盖。**
3. **支线任务（含时限失败态）**：如克莉丝汀想报复金手指帮，玩家长期不响应则她带走物资独自离开、日后在其据点发现尸体（详见下文"首条成形支线：克莉丝汀"）。
4. **聚餐气泡交流**：黎明/黄昏两次强制聚餐时（见 §4 相位机），角色以头顶气泡交流——推进支线（如克莉丝汀反复请求清剿金手指帮，详见下文"首条成形支线：克莉丝汀"）、提营地需求（诺蒂想要书柜/书桌/沙发，满足后 +5% 阅读速度小任务奖励）。气泡由事件驱动 + 关系变化自动触发，替代原先的信件作为支线入口。
5. 角色死亡后，其支线相应**中断或转向**。

### 首条成形支线：克莉丝汀（教学关起点）

克莉丝汀是第一个把"招募→成长→支线（含时限失败态）→聚餐气泡→死亡/转向"五件套走完整的角色实例，同时她的登场即游戏的**教学关**。

1. **教学关触发（招募）**：第 2~3 天（拟定待调），一伙流浪劫掠者袭营，作为**教学关**教玩家防御与战斗（袭营规则见 §5"守卫防御战"）。克莉丝汀混在劫掠者之中，**阵中反水**——起手她与劫掠者同阵营。
2. **反水触发与台词**：教学关战斗中，当**任一劫掠者受伤较重**或**克莉丝汀自己受伤**（先到者触发，阈值拟定待调）→ 她喊出台词 **"杀死这些劫掠者！我是好人！"** → **立即反水**，从劫掠者阵营切为助玩家、转而攻击其余劫掠者。
   - **控制权流转**：反水后她**不是玩家可控**——切阵营后仍为**自动战斗的盟友 AI**（自动攻击劫掠者、助玩家，但玩家操控不了）；**只有战斗结束且玩家选择"收留"后**，她才转为**玩家可控的营地幸存者**；放逐/处决则离开或死亡。即状态流转：劫掠者 AI（攻击幸存者）→[反水]→盟友 AI（自动打劫掠者·不可控）→[战斗结束+收留]→可控幸存者。
3. **生死后果**：教学关袭击整体走守卫防御战的 RaidResolution 胜负/伤亡判定（见 §5"守卫防御战"）。克莉丝汀**作为参战单位不受特殊保护——她可能战死；一旦战死，整条克莉丝汀支线不触发**（黑暗向、有意为之，勿当平衡问题修复）。
4. **战后抉择**：克莉丝汀存活时，玩家可选 **收留 / 放逐 / 处决**，各有分支。**表现差异**：放逐 = 她走向门外淡出消失；处决 = 原地留下血渍、单位消失。
5. **收留后·支线推进**：聚餐环节她以剧情气泡**反复（3 次，拟定待调）请求清剿"金手指帮"**（气泡机制见 §4 相位机 / §8 上文聚餐气泡）。**每次请求在那一餐后弹出抉择面板（答应出兵 / 暂不）**：选"答应"→置出兵意向、此后不再逼问；选"暂不"→计一次，累计到 3 次触发下方时限失败态。请求台词为占位草稿，最终由用户手写（可一句带过）。背景：她曾被金手指帮轮奸，好不容易逃出，后加入一个劫掠者团伙勉强活命。
6. **时限失败态**：累计 3 次"暂不"后，她不立即离开，而是在**下一次昼夜交替时自愿离开营地去复仇**（非即时，支线失败/转向）。
7. **结局·环境叙事**：分处两处探索目的地（异地）——
   - **金手指帮根据地**（清剿后）：此处按支线走向可发现**两具不同的尸体**（关卡各铺一个发现点，复仇线两点同现、先帮众后她本人）：
     - **通用遗体（各分支都可见）+ 日记A**：一名**被克莉丝汀反杀的金手指帮成员**（男性帮众）。当年金手指帮对被掳的克莉丝汀"手软"，她趁隙反杀了其中一名帮众，其余帮众才把她彻底绑起来。这具尸体在克莉丝汀被绑走前就已存在、时间线上早于本次探索，**故不按她的去向门控**——无论玩家是否收留她、她是否去复仇，都能见到（死状为利器近身反杀、现场有拼死反抗痕迹）。**日记A（帮众自白：互助求生→参与暴行、"金手指"命名由来）即拾于这具帮众尸体旁**，故各分支都能取得。
     - **复仇线专有遗体**：仅当克莉丝汀三拒后独走复仇（`christine_left_for_revenge`）时，另点名发现**克莉丝汀本人的尸体**——她遭金手指帮极端暴行致死（尸体状态成因见 §3"派系：金手指帮与哥顿"）。此为与通用遗体不同的**另一具**。
   - **守望者森林小屋**：发现**哥顿上吊尸（门口老树上）+ 日记B**——帮主哥顿早已独自上吊自杀；日记B 揭其身世（强势母亲、丧尸母亲当面弑父、以暴掩懦、看透后自杀），成因见 §3。
   - **日记为可拾取 lore 物品**：日记A / 日记B 均可在现场拾取、带回营地阅读。

**教学关实现口径（拟定待调）**：第 2 天夜晚一次性触发；劫掠者为携武器的人类敌人；收留后克莉丝汀转为营地幸存者，聚餐请求台词由剧情 flag 触发。

### 专属效果系统（authored·替代通用技能·已落地）

角色**没有通用技能**；每角色的特长以 **authored 专属效果**承载（与关系 / 性格同属手写剧情，不做程序化数值系统）。

- **形态**：每角色的专属效果分 **1 / 2 / 3 级**，**每角色的升级条件各不相同**（手写）。系统只做"**按条件播放 / 结算 authored 效果**"的框架，不程序化生成效果或成长曲线。
- **首个专属效果：诺蒂·"书虫"**（阅读加成；升级靠累计阅读时间）——
  - **1 级**：自身阅读速度 **+25%**。
  - **2 级**：自身阅读速度 **+50%**。
  - **3 级**：自身阅读速度 **+50%**，且**全营所有角色阅读速度 +25%**（诺蒂自己叠加为 **×1.75**）。
  - **升级条件**：**累计阅读时间**达阈值（拟定待调，如 **48h → 2 级、120h → 3 级**）；累计随"读书活动"增长（见 §7"读书活动"）。
  - **读速合成（加法）**：`基础读速 ×（1 + 自身加成 + 全营加成汇总）× 座位系数`——全营加成来自任意达 3 级"书虫"角色（多来源汇总相加）；座位系数见 §7"读书活动"。
- **口径覆盖**：原 §8"行为驱动三级成长"举例的旧占位数值 **+15 / 35 / 50%** 与全员 **+5%**，统一以本节 **+25 / 50 / 75%**（3 级 = 自身 +50% ＋ 全营 +25%）覆盖。

### 伤残与叙事

- 战斗中的切除/伤残不是一次性的战报事件，而是**长期跟随角色的伤痕**：断手断腿带来永久的操作/移动能力损失（数值见 §5"伤残能力与假肢系统"），角色以残疾状态继续在营地生活与劳作。
- **假肢**是营地经营对残疾的回应——从简陋的木手木腿到稀有的仿生假肢，部分恢复能力，也在角色身上留下"这一路付出了什么"的可见叙事。装上假肢后该部位不再被切除，是一种"定格的代价"。
- 伤残叙事与角色支线、聚餐气泡交织：残疾角色的处境、旁人的反应，都是营地故事的一部分。

### 解锁与生产

- 搜集的书籍可阅读解锁**蓝图**（如枪械制造手册），夜间生产制造装备与工事。**制作 / 配方 / 工具 / 武器改装的完整规则见 §7 制作与配方系统**——书籍解锁按**制作者本人已读**判定（如骨刀需读完《野外生存指南》）；角色无通用技能，配方门槛即"读过对应的书"。

## 9. 失败与死亡

- 任何幸存者可在探索、防御战、剧情中永久死亡；重伤可致永久伤残（切除/骨折后遗症）。
- 营地全灭 = 游戏结束（**通用失败态**，非 §3 三条正史结局之一——玩家未走到任何结局就团灭的兜底态）。

## 10. 开发优先级（支柱排序）

1. **战斗结算引擎**（武器/护甲/部位/逐层判定）+ 蒙特卡洛数值模拟器 —— 差异化核心，风险最集中。
2. **昼夜循环骨架**：世界地图选点 → 关卡探索 → 回营 → 夜晚实时+排班+防御战。
3. **角色与叙事**：先做 2~3 个角色（诺蒂、克莉丝汀）打通"招募→成长→支线→失败态"全链路。
4. 生产/蓝图、章节装置（温度/辐射）、人类袭击 AI、主线结局 —— 依次后置。

## 11. 明确不做（YAGNI）

- 程序化生成地图/角色/任务。
- 全局温度、心情模拟系统。（2026-07-06 更新：辐射已从 YAGNI 挪为**待实现规划**——拟做成全局白天系统，方向见 §4/§5，但代码尚未落地，尚非已实现功能。）
- 多人、模组支持、长线沙盒平衡。
- 国内平台过审适配。
