# Dead Signal · 本地数值 wiki

一个跑在本机的网页，用来看和改游戏里所有物品的数值。**它取代了 xlsx**。

带分区、筛选、排序、行内编辑；改完点「保存」直接写回仓库里的 JSON 文件。纯静态网页 + 一个 python 小服务，
零第三方依赖、离线可用、不联网、不碰数据库。

---

## 怎么用（两条命令）

```bash
# 1. 从代码里把数值抽出来（第一次用，或者代码改过之后）
export DOTNET_ROOT=$HOME/.dotnet
~/.dotnet/dotnet run --project tools/WikiExtract

# 2. 起服务
python3 tools/wiki-serve.py
```

打开它打印的地址（默认 <http://127.0.0.1:8787/>）。端口被占就 `--port 8788`。

页面上：

- **左侧**切分区（武器 / 护甲服装 / 材料 / 配方 / …）
- **搜索框**按任意列的内容过滤；**彩色小钮**按种类筛（点第二下取消）
- **点列头**排序，再点一下反序，第三下取消
- **点格子**直接改；改过的格子会亮起琥珀色，改过的行会变暗黄底
- **保存**（或 `Cmd/Ctrl+S`）把改动写回 `docs/wiki/data/*.json`，右下角会弹提示说存了几个分区
- 「显示内部 id 列」勾上才看得到那些置灰的 `内部 id` / `代码位置` —— **别改它们**，agent 靠它们定位代码

没起服务、直接双击 `index.html` 也能打开，但那是**只读模式**（浏览器不许网页写本地文件）。
那种情况下可以用「导出 JSON」把改动下载下来，再覆盖回 `docs/wiki/data/`。

## 新增和删除

- **新增**：点「+ 新增一条」→ 新行加在**表格末尾**（武器一律追加末尾不插队，插到中间会打乱 Sim 的随机流）→
  填好「名称」和置灰的「内部 id」→ 保存。内部 id 不能空、不能重复、不能有中文，否则存不进去（会拦下来告诉你哪儿不对）。
- **删除**：把鼠标移到某一行，行尾会出现一个 `×` → 点它 → 二次确认 → 那一行变成**划掉的灰行**（再点 `↩` 可以撤销）。

> ⚠️ **新增和删除都不会自动进游戏。**
> 在网页上删掉一把武器，游戏里那把枪**还在**——这里改的是设计表，不是代码。
> 所以这两种行会一直带着「**待同步进代码**」的标记，直到 agent 把改动落进 C#。

删除为什么是"划掉"而不是直接消失？因为如果直接从 JSON 里抹掉，下次重跑抽取器又会从代码里把它捡回来——
你的删除就"自己撤销"了。留一行划掉的墓碑，才能一直提醒 agent「这条该从 C# 里删掉」。
等它真从代码里删了，重跑时这行就自动消失了。

## 改完之后：怎么让改动进游戏

**这个网页只改 JSON，不改代码。** JSON 是设计表，C# 才是游戏真正读的东西。

所以流程是：

1. 你在网页上改数值 → 点保存 → `docs/wiki/data/*.json` 变了
2. **跟 agent 说一声**：「wiki 上改了武器和配方，同步进代码」
3. agent 把 JSON 的改动落到对应的 C#（靠每条记录的 `内部 id` / `代码位置` 定位），跑测试门禁（`dotnet test` 全绿 + Godot 构建 0 警告 0 错误），必要时用 Sim 复验平衡没被改坏

反过来，**代码里加了新东西**（新武器、新材料、新配方）之后，重跑一次抽取器，新条目自己就会出现在网页上：

```bash
~/.dotnet/dotnet run --project tools/WikiExtract
```

抽取器是**反射遍历 catalog** 的，不是手抄的——往 `WeaponTable` 里加一把枪，它自己就捡得到。

### 重跑抽取器**不会**弄丢你的改动

**表赢代码**：你在网页上改的、加的、删的，重跑抽取器都会原样保留。它只做四件事：

1. 代码里新增的条目 → 补进表里
2. 表里已有的条目 → **值一律以表为准**；和代码现值对不上的，在命令行列成一份「待同步」清单
3. 你新增的条目 → 保留，继续标「新增·待同步进代码」
4. 你删掉的条目 → 保留那行划掉的墓碑，继续标「删除·待同步进代码」

跑完它会把所有「表 ≠ 代码」的地方打印出来，**带上该改哪个文件哪一行**，比如：

```
⚠️ 有 2 处「表 ≠ 代码」——表里的内容已原样保留，需要同步进 C#：
  [删除待同步] 武器·草叉（id=Pitchfork）在表里被删了，代码里还有
              ⇒ 把它从 src/DeadSignal.Combat/WeaponTable.cs :: WeaponTable.Pitchfork() 删掉
  [新增待同步] 武器·铁撬棍（id=crowbar）表里有、代码里没有 ⇒ 该在 C# 里把它建出来
```

这份清单就是给 agent 的工单。

> 想把表推倒重来、完全按代码重建（**丢弃表里的全部改动**）：`~/.dotnet/dotnet run --project tools/WikiExtract -- --reset`

## 数据流（一张图）

```
   C# 代码  ──────► tools/WikiExtract ──────►  docs/wiki/data/*.json  ──────►  网页
（唯一事实源）        （反射抽取，可重跑）          （设计表，你改这个）        （看 + 改）
      ▲                                                  │
      │                                                  │ 点「保存」PUT 回文件
      └──────────  agent 手工回写 + 跑测试门禁  ◄─────────┘
```

**C# 始终是唯一事实源**——游戏运行时不读这些 JSON。这是刻意的：数值改动必须过测试门禁和 Sim 复验，
不能让一个手滑的格子直接进游戏。

## 「每秒伤害」这两列

改了伤害或攻击间隔，DPS 会**立刻跟着变**——这就是它存在的意义。

> ⚠️ **它是杀伤力天花板，不是真实战力。** 按无甲、贴脸、无限弹药、单挑算出来的，
> **不含**护甲、距离衰减、开枪招来的怪、子弹够不够打、一枪撂倒几只。
> 枪的真实战力其实由**弹药供给**决定，而供给不在这个数字里。

- **不可双持的武器**，「双持每秒伤害」显示 **—**（不是 0）。
- **霰弹枪**按 8 颗弹丸**全中**算 ⇒ 那是理论上限，实战打不到（每颗弹丸独立选部位、独立判定，打披甲目标会被挡掉几颗）。
- **双持 ≠ 常规 × 1.4**：双持的惩罚只落在**冷却**上，连发那一段不受罚 ⇒ 连发武器双持反而更划算。

**公式的唯一事实源是引擎**（`src/DeadSignal.Combat/WeaponDps.cs`），网页只做实时预览。
页面加载时会拿自己算的和引擎算的**逐行对账**——万一哪天引擎改了规则而网页没跟上，
页面会**弹横幅警告并停用预览**，绝不会悄悄给你一个错的数字。

## 图标

约定：`godot/assets/items/<分区id>/<图标文件名>.png`。图标文件名在每行的置灰内部列里（小写蛇形，如 `improvised_hunting_gun`）。

- 缺图的条目画一个虚线问号框，**不会**报 404（服务的 `GET /api/icons` 出一份现有图标清单，网页只给真有图的条目发请求）
- 要知道还缺哪些图：看 `docs/wiki/data/icon-manifest.json`（抽取器自动生成的「该画哪些图」清单）

## 加一个新分区

页面本身**不认识任何具体分区**——列、类型、配色、筛选、编辑控件全部由 JSON 里的 `columns` 元数据生成。
所以加分区不用碰 HTML/JS：

1. 在 `tools/WikiExtract/Program.cs` 的 `Main()` 里，往 `categories` 列表加一个方法（照抄 `Weapons()` 或 `Books()`）
2. 重跑抽取器 —— `index.json` / `bundle.js` / 图标清单都是自动补的

列的 `type` 只有 6 种：`text` / `longtext` / `number` / `bool` / `chip`，加上 `primary`（中文名主键列，冻结在左侧）、
`readonly`（代码推导出来的，改了没用）、`internal`（置灰勿改，默认隐藏）、`hint`（鼠标悬停的提示）四个修饰。

> 给用户看的列一律**人话中文**：类名、英文 id、引擎术语只许出现在 `internal` 的置灰列里。

## 🔴 给 agent：别杀这个服务

`tools/wiki-serve.py` 是**长期运行的工具服务**，不是谁的临时测试进程——**用户可能正开着页面在改数值**。

- ❌ **不许 `pkill -f wiki-serve.py`**（哪怕那个进程是你自己起的）。杀了它，用户点保存就是 `Failed to fetch`，而**网页上没保存的改动只在内存里，服务一没就全丢**。这事已经发生过一次，用户当时改了 7 处。
- ✅ 要验证保存链路 → **复用已经在跑的那个**（`curl` 打 <http://127.0.0.1:8787/> 就行），别自己起也别关。
- ✅ 非要起一个隔离的测试服务 → **换端口**（如 `--port 8792`），收尾只杀自己那个：`pkill -f "wiki-serve.py --port 8792"`。

收尾清理只删**自己造的文件**（截图、临时目录、`__pycache__`），**不碰进程**。

## 那个小服务

`tools/wiki-serve.py`，python3 标准库写的，只做三件事：下发 `docs/wiki/` 的静态文件、
把 `godot/assets/` 挂在 `/assets/` 下供图标读取、接受 `PUT /data/<名字>.json` 写回 JSON。

写入是它唯一有副作用的操作，所以门槛卡死：只认 `/data/<名字>.json`（名字必须是 `[a-z0-9_-]+`，
连点和斜杠都进不来）、落盘前 realpath 复核仍在 `data/` 内、请求体必须是结构完整的分区表（不接受把一张表清空）、
先写临时文件再原子替换、只监听 `127.0.0.1`。
