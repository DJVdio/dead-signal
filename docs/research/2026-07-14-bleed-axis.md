# 流血轴复验（T53 · 用户拍板「方案 A：先跑 Sim 复验」的前置条件）

> **历史快照**：本文仅保留当日仿真结果。当前武器、护甲及其他配置数值一律以 Wiki 配置表为准；本文表外的静态数值说明不代表当前配置。

**用户设计口径（事实源）**：「我的设计是，锯齿剑刃是用来对付无甲或者轻甲目标，边打边跑」

⇒ 主判据 = **梯度**：对无甲/轻甲要明显强于原厂刃，对中甲/重甲要明显弱于原厂刃。
穿透惩罚是梯度的历史仿真输入之一：划不进甲 ⇒ 没有伤口 ⇒ 没有流血。

口径：每格 3,000 个种子；攻方 = 幸存者（中甲：皮夹克+布衣）；
守方人类 = 持棍棒劫掠者（甲随档位）。**「流血致死占比」= 胜场中敌方被放干致死的比例**。

> ⚠️ **Sim 测不出「边打边跑」**：`Duel` 是 1v1、无距离、无走位的站桩对砍。
> 下面所有胜率都是**站桩**胜率 —— 恰恰是锯齿剑最不该被评估的场景。风筝战术的真实强度只能实机校准；
> §3 的放干耗时表是给风筝可行性的**边界估算**（算术），不是战术强度。

## §4 穿透惩罚扫描 —— 要多大的惩罚，梯度才成立？

§1 证明 **−20% 穿透惩罚下，锯齿刃是无脑上位替代，且对重甲收益最大**（梯度是反的）。

**为什么加大穿透惩罚是对的解法**：**无甲目标身上一层护甲都没有 ⇒ 穿透在结算里根本不被读取**。
⇒ 穿透惩罚对无甲**零代价**、对重甲**全额代价** —— 这正是用户要的形状。问题只是 −20% 太小。

下表：长剑（原厂穿透 24%），流血固定 +40%，只扫穿透惩罚。**胜率差 = 锯齿 − 原厂**（正=更强）。

### 读法 —— 穿透这根杠杆对四档甲的「抓地力」完全不同

1. **无甲列：任何惩罚下都是 0.0%**。无甲目标身上一层护甲都没有 ⇒ **穿透在结算里根本不被读取**。
   ⇒ 穿透惩罚对无甲是**完全免费**的。（胜率本就 99.2% 已饱和 ⇒ 对无甲的收益**不可能**体现在胜率上，
   只体现在**耗时**：见 Δ耗时列，以及 §3 的放干耗时。）
2. **丧尸列：任何惩罚下都是 +3.1%，纹丝不动**。丧尸腐皮太薄（锐防 1.5）⇒ **穿透对丧尸也没有抓地力**。
   ⇒ **没有任何穿透惩罚能让锯齿刃对丧尸变差。** 而丧尸本就是「轻目标」= 用户设计的正靶 ⇒ **这条是对的，不用管。**
3. **重甲列：这是唯一需要修的一格**，也是穿透**唯一有抓地力**的一格。零点在 **≈ −50%**。

⇒ **结论：穿透惩罚是一根「只打重甲、不碰其它三档」的精准杠杆** —— 正是用户设计需要的形状。

> **注**：`MaxWoundsPerPart = 3` ⇒ 单部位最多 3 处伤口。上表是「全部命中致命部位」的最快情形；
> 打中手/脚（权重 0.5）或指/趾（0.2）只会更慢，且**非致命伤口永远放不干**
> （`NonLethalBloodFloorRatio = 50%` 是硬下限——划破手指流不死人）。

> ✅ **链路已核实：跑开之后流血还在走。** `Actor._PhysicsProcess`（godot/scripts/Actor.cs:378）
> 每物理帧对**任何有伤口的活着的 actor** 调 `Body.TickBleed(delta)`，**不以「在战斗中」或「距离」为条件** ⇒
> 用户的「边打边跑」在消费层是**真的成立**的，不是纸面设定。

## §6 🔴 伤口触发诊断 —— 回答「是不是对无甲没那么容易挂流血、对重甲太容易挂」

**用户假设**：「我觉得问题出在流血触发机制，是不是对于无甲目标没那么容易挂流血，对重甲目标太容易挂流血了？」

**结论：假设不成立，而且实际情况正好相反 —— 护甲让流血【更难】挂上，不是更容易。**

先看代码（`Effects.cs` 的流血门，这是伤口诞生的**唯一**入口）：

```csharp
bool sharp = result.FinalDamageType == DamageType.Sharp;  // 锐器【抵达】才有资格
if (sharp && dmg > 0 && !severed)                          // ← 伤害被挡光(dmg==0) ⇒ 连 roll 都不抽
{
    double p = Clamp01Cap(BleedK * dmg / maxHp, BleedCap); // ← 概率【正比于实际打进去的伤害】
    if (rng.Range(0, 1) < p) body.RegisterBleed(...);      //   BleedK=1.0, BleedCap=0.90
}
```

⇒ **护甲从三条路同时压制流血**（不是助长）：
1. **完全挡下（dmg == 0）⇒ 一处伤口都不记**，连随机都不抽。**用户担心的「挡光了还留个口子」在代码里不存在。**
2. **半穿（Half）⇒ 锐器被降解成钝伤**（`currentType = Blunt`）⇒ `sharp == false` ⇒ **完全丧失流血资格**。
3. 即使打进去了，**伤害更低 ⇒ p 更低**（p 正比于 dmg）。

实测（长剑 × 200,000 次单发命中，命中部位按体积权重随机）：

| 目标 | **每刀挂流血率** | 平均实际伤害 | 完全挡下率 | 锐转钝(降解)率 |
|---|---|---|---|---|
| 无甲 | **58.0 %** | 9.01 | 0.0 % | 0.0 % |
| 轻甲·长袖布衣 | **57.7 %** | 8.98 | 0.0 % | 1.1 % |
| 中甲·皮夹克+布衣 | **50.3 %** | 7.75 | 1.1 % | 8.7 % |
| 重甲·板甲+外套+布衣 | **25.3 %** | 4.52 | 38.9 % | 18.3 % |

⇒ **无甲的每刀挂流血率是全表最高的，重甲最低。触发机制没有 bug，方向完全正确。**

### 那「重甲受益最大」到底是怎么来的？—— **边际效用，数学上的必然**

锯齿刃的 +40% 是**乘在流血这一项上**的。它值多少钱，取决于**流血在这场击杀里占多大比重**：

| 目标 | 每刀挂流血率 | 战斗时长 | **流血致死占比** | +40% 的边际收益 |
|---|---|---|---|---|
| 无甲 | **最高** | 24.0s（短） | 45.5% | **小** —— 直伤本来就够杀，流血是多余的 |
| 重甲 | **最低** | 55.1s（长） | **78.2%** | **大** —— 直伤砍不动，杀人主要靠放血 |

**两个方向相反的效应**：护甲让流血**更难挂上**（每刀挂血率↓），但也让**直接伤害更砍不动**、
于是**仗打得更久**（24s → 55s，流血有 2.3 倍的时间去积分），且**直伤这条腿被打断** ⇒
**流血在总击杀里的占比反而从 45.5% 升到 78.2%**。
⇒ **后者压倒前者** ⇒ 给流血 +40%，对重甲的边际收益自然最大。**这不是 bug，是「乘在一个占比更大的项上」的算术。**

⚠️ 还有一条**测量假象**：打无甲的胜率**本来就是 99.2%（饱和）**——
**再强也没处涨** ⇒ 「对无甲更强」**在胜率这把尺子上根本量不出来**。
它的收益只体现在**节奏**（击杀 24.0s → 22.0s）与**风筝**（放干耗时，§3）—— 而**风筝正是 Sim 测不了的东西**。

### 顺带核实：3 处/部位的伤口上限有没有在无甲身上「打满饱和」？

`MaxWoundsPerPart = 3` 是**单部位**上限，而命中部位按体积权重**分散**在全身（躯干/头/四肢…），
**不是全身 3 处上限**。要在同一个部位吃满 3 刀才会饱和；无甲仗只打 24s（长剑 3.2s 冷却 ⇒ 约 7 刀），
这 7 刀还要分散到多个部位 ⇒ **单部位打满 3 处的情形很少，上限不是这里的瓶颈**。
（且真饱和了也只会**削弱**无甲侧的流血，与「重甲受益更大」同向，不改变结论。）
