# Dead Signal（死信号）

俯视角 2D 丧尸生存经营游戏 · 中短流程剧情向 · 对标《这是我的战争》 · Godot 4.7 + C# · 像素风

个人项目，不考虑发行。

## 概念

丧尸**白天休眠、夜晚游荡**。你经营一处庇护所：白天派 ≤3 人小队去手工设计的地点搜刮（人类劫掠者只在白天活动——主力出门时老家可能被抄）；夜晚实时度过，排班站岗/生产/读书，丧尸袭营时打实时 + 战术暂停的防御战。

营地电台能收到军方救援广播，但只能收不能发——主线是远征广播台修好它。呼叫救援与否，是两个都不轻松的结局。全员永久死亡，无主角；每个幸存者都有手写的招募剧情、支线（含时限失败态）。

## 已实现系统

> 数值均为「拟定待调」占位，规则形态才是设计重点。**单一事实源：`docs/superpowers/specs/2026-07-04-dead-signal-design.md`**。

### 战斗引擎（完成度最高的部分，纯 C# 零依赖）

- **部位树 + 独立 HP**：细到左右眼/鼻/下巴；头/颈/躯干归零致死，四肢致残，切除有连带（断臂失手）
- **逐层护甲三段判定**：连续小数 roll，全伤/半伤转钝/挡下；锐器被降解后无任何状态效果。护甲按部位覆盖（哪片甲护哪些部位）
- **效果系统**：流血（扣储血量→轻/中/重度昏迷→出血致死）、切除、震荡（隔甲生效）、骨折（天然钝器专属）、0HP 部位钝伤磨损→永久损毁
- **命中模型**：无命中率概念——近战必中，远程按误差角锥形弹道 + 射程/距离衰减；贴脸枪托近战
- **持握 GripMode**：单手 / 一把单手改双手握（+15% 攻速）/ 双持两把单手（×0.70 攻速、远程误差角 ×1.25）/ 双手武器占两手；断手→该手不能持械
- **负重分段惩罚**、攻击冷却状态机；克制三角：锐克无甲、甲克锐、钝克甲

### 装备 / 持械

- **11 槽穿戴系统**：占用/校验/断肢禁装/替换/卸装，聚合「哪些部位被护甲覆盖」供战斗层只读消费
- **左右手持械**：主手/副手各持一把，推导 GripMode 喂给引擎系数
- 装备 UI（角色面板），断肢与穿戴/持械联动

### 制作 / 技能 / 材料 / 改装

- **材料目录**：配方消耗的基础材料数据表，落地为库存物品（`Item.Material`）
- **幸存者技能 SkillSet**：纺织/机械/化学 + 医疗/烹饪/战斗，经验累积升级、门槛查询
- **配方 + 工作台三工具槽**：配方 = 材料 + 需要工具 + 制作者门槛（读过的书 / 技能）→ 产物（+ 回喂经验）；卡尺/锯片/烧杯装上才解锁对应品类。`CraftingLogic` 出纯函数「能不能做 + 缺什么 / 扣什么产什么」，由调用方去库存实扣实产
- **武器改装 WeaponMod**：按武器部位（枪托/枪管/枪口…）挂改装件，同部位不叠、异部位可叠

### 营地 / 防御 / 阵营

- **平地四面围栏 + 南北门**：围栏/大门有可破坏 HP、建造等级（等级→血量）
- **守卫岗位**：哨塔（+射程/视野、围栏抵挡远程）/ 屋顶平台 / 暗哨（先手一击）
- **袭营破防**：敌人够得着就砸墙、够不着就贴近的纯逻辑地基（择结构按边缘距离）
- **三方阵营 + 敌对矩阵**：幸存者 / 丧尸 / 劫掠者；`Factions.IsHostile` 统一裁定敌我，运行时可切阵营 / 换择敌池（克莉丝汀反水即靠此）
- **劫掠者 AI**：携武器的人类敌人，昼夜皆活动，择敌走可替换候选池 + 敌对矩阵

### 物品 / 库存 / 搜刮 / 探索 / 剧情

- **营地共享库存**：食物/书/护甲/武器/材料五类物品的增删查（不做每人背包）
- **搜刮 / 容器 loot / 阅读**：探索关卡里翻容器、读书涨技能
- **探索到根据地 + 发现点日记**：克莉丝汀线结局（金手指帮根据地的尸体 + 日记 A、守望者森林小屋的上吊尸 + 日记 B）
- **剧情框架**：`StoryFlags` 字符串态状态机（引擎无剧情整数字段），按条件播放手写剧情——克莉丝汀教学关（第 2 夜劫掠者袭击 / 反水 / 抉择 / 招募）、收留·放逐·处决抉择面板
- **昼夜循环 / 饥饿 / 聚餐气泡**、游戏结束判定（全体幸存者死亡→暂停 + 模态）

## 项目结构

```
src/DeadSignal.Combat/   战斗规则引擎（纯 C# 类库，零依赖；空间问题不进这里）
src/DeadSignal.Sim/      蒙特卡洛模拟器 + 逐回合对决战报（拉数值表校准方向）
tests/                   xUnit 单测（38 个文件，428 个 [Fact]/[Theory]；含变异验证）
godot/                   Godot 4.7 工程 —— RimWorld 式操控的可玩场景
  scripts/               消费层：营地/战斗表现/UI/装备/制作/守卫/剧情 编排
  data/                  数据驱动配置（daynight/camp/meal_bubbles…）
docs/superpowers/specs/  设计文档（单一事实源）
docs/research/           数值基线报告 / 对决战报 / 美术可行性调研
```

**分层原则**：`DeadSignal.Combat` 只出纯函数规则（引擎，零依赖）。营地/装备/制作/守卫等系统的**纯逻辑**放在 `godot/scripts` 里不引 Godot 类型的文件（`ApparelSlots` / `WeaponLoadout` / `SkillSet` / `Recipe` / `CraftingLogic` / `Factions` / `BreachLogic` / `GameOverCondition` …），以 **Link 方式**编进 `DeadSignal.Combat.Tests` 单测覆盖；空间执行（寻路/碰撞/弹道飞行/伤害施加）才写进 Godot 运行时层。随机一律走可注入的 `IRandomSource`。

## 构建 / 测试 / 运行

.NET 8 SDK 装在用户目录、**不在 PATH**，一律用 `~/.dotnet/dotnet` 并设 `DOTNET_ROOT`。Godot 用 4.7 .NET 版。

```sh
export DOTNET_ROOT=$HOME/.dotnet

# 构建（根 sln = 引擎 + 模拟器 + 测试，不含 godot 工程）
~/.dotnet/dotnet build DeadSignal.sln -c Release
~/.dotnet/dotnet build godot/DeadSignal.Godot.csproj   # godot 工程单独构建

# 测试（全绿是硬门禁；改战斗规则必须先红后绿）
~/.dotnet/dotnet test

# 模拟器（无参 = 聚合蒙特卡洛数值表；duel [路径] = 逐回合对决战报）
~/.dotnet/dotnet run --project src/DeadSignal.Sim

# 跑游戏（headless 跑 C# 需另 export PATH="$HOME/.dotnet:$PATH"）
/Applications/Godot_mono.app/Contents/MacOS/Godot --path godot      # 加 -e 进编辑器
```

试玩操控：左键选人 / 右键指令 / 空格暂停 / 1·2·3 变速 / T 跳昼夜。
