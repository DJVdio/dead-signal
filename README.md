# Dead Signal（死信号）

俯视角 2D 丧尸生存经营游戏 · 中短流程剧情向 · Godot 4 + C#

## 概念

丧尸**白天休眠、夜晚游荡**。你经营一处庇护所：白天派 ≤3 人小队去手工设计的地点搜刮（人类劫掠者只在白天活动——主力出门时老家可能被抄）；夜晚实时度过，排班站岗/生产/读书，丧尸袭营时打实时+战术暂停的防御战。

营地电台能收到军方救援广播，但只能收不能发——主线是远征广播台修好它。呼叫救援与否，是两个都不轻松的结局。全员永久死亡，无主角；每个幸存者都有手写的招募剧情、支线（含时限失败态）与信件。

## 战斗系统（本仓库当前完成度最高的部分）

- **部位树 + 独立 HP**：细到左右眼/鼻/下巴；头/颈/躯干归零致死，四肢致残，切除有连带（断臂失手）
- **逐层护甲三段判定**：连续小数 roll，全伤/半伤转钝/挡下；锐器被降解后无任何状态效果
- **效果系统**：流血（扣储血量→轻/中/重度昏迷→出血致死）、切除、震荡（隔甲生效）、骨折（天然钝器专属）、0HP 部位钝伤磨损→永久损毁
- **命中模型**：无命中率概念——近战必中，远程按误差角锥形弹道；双持攻速 70%（远程另 +25% 误差角）
- **负重分段惩罚**，克制三角：锐克无甲、甲克锐、钝克甲

全部规则由 61 个单元测试锁定（含变异验证），详见 `docs/superpowers/specs/2026-07-04-dead-signal-design.md`。

## 构建与运行

需要 .NET 8 SDK 与 Godot 4.7（.NET 版）。

```sh
dotnet build DeadSignal.sln -c Release   # 引擎+模拟器+测试
dotnet test                              # 61 tests
dotnet run --project src/DeadSignal.Sim  # 聚合数值表；加 `duel` 出对决战报
Godot --path godot                       # 试玩最小场景（左键选人/右键指令/空格暂停/1·2·3变速/T跳昼夜）
```
