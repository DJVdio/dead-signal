# Dead Signal（死信号）

俯视角 2D 丧尸生存经营游戏 · 中短流程剧情向 · 对标《这是我的战争》 · Godot 4.7 + C# · 像素风

个人项目，不考虑发行。

## 概念

丧尸**白天休眠、夜晚游荡**。你经营一处庇护所：白天派 ≤3 人小队去手工设计的地点搜刮；夜晚实时度过，排班站岗/生产/读书，丧尸或人类劫掠者袭营时打实时 + 战术暂停的防御战。全员永久死亡、无主角；每个幸存者都有手写的招募剧情与支线，还有一条推动玩家行动的全局压力驱动主线。

战斗与潜行的骨架是**声音、光线与后勤**：枪很强，但枪响会把屏幕外的东西全拽过来，而且子弹造一发少一发；弓很弱，但它比你自己的脚步声还安静，箭还能捡回来。

> **剧情/世界观/结局细节一律不写进本文档**，以免剧透。完整叙事与玩法规则见设计文档（**含剧透**）：`docs/superpowers/specs/2026-07-04-dead-signal-design.md`。README 只讲**工程与玩法机制框架**。

## 核心循环

- **白天（丧尸休眠）**：从世界地图选点，派 ≤3 人小队进手工关卡搜刮物资、遭遇剧情、招募幸存者；天黑前必须回营。
- **夜晚（丧尸游荡）**：实时度过（1x/3x/8x 变速，事件自动拉回 1x），排班站岗/生产/修理/读书；**常规袭营只发生在夜里**（丧尸与人类劫掠者都夜袭），进实时+战术暂停的防御战。
- **硬性双班作息**：昼班（探险）与夜班（站岗/生产）分开，非本班时段强制睡、不可操作，人手永远紧缺。每日核心权衡由此而来——**谁编进探险队，夜里的守备名册就相应变薄**，越可能被迫叫醒睡着的人应战（并吃次相位疲劳惩罚）。
- **全局时间压力**：存在一条推动玩家推进主线、不能无限拖延的世界级压力（具体形态见设计文档）。

## 已实现系统

> 数值均为「拟定待调」占位，规则形态才是设计重点。以下只列机制；机制的剧情后果见设计文档。

### 战斗引擎（完成度最高的部分，纯 C# 零依赖 · 设计文档 §5）

- **部位树 + 独立 HP**：细到左右眼/鼻/下巴；头/颈/躯干归零致死，四肢致残，切除有连带（断臂失手）。
- **逐层护甲三段判定**：连续小数 roll，全伤/半伤转钝/挡下；护甲按部位覆盖（哪片甲护哪些部位）。全程小数、不取整。
- **多弹丸齐射**：霰弹枪一发打出 8 颗弹丸，**每颗独立选命中部位、独立走三段判定**——于是"打披甲的挡下大半、打光身的糊一脸"是从弹丸层面涌现的，不是配的系数。
- **效果系统**：流血（扣储血量→轻/中/重度昏迷→出血致死）、切除、震荡（隔甲生效）、骨折（天然钝器专属）、0HP 部位钝伤磨损→永久损毁；**命中即使未破防也施加短暂减速**（RimWorld stagger 式全角色通用轻量效果——只降移速、不打断攻击冷却，与震荡严格区分）。护甲判定之后还有一层**乘算伤害减免**接口，供角色专属效果消费。
- **命中模型**：无命中率概念——近战必中，远程按误差角锥形弹道 + 射程/距离衰减；贴脸枪托近战；枪械冷却/连发节奏。
- **持握 GripMode**：单手 / 双手握一把（**无攻速加成**，与单手同基线）/ 双持两把单手（×0.70 攻速、误差角 ×1.25）/ **强制双手的武器占满两手**（武器自身的静态标识，与"当前怎么握"是两回事——占了两手就不能再拿第二把武器，也不能同时举火把）；断手→该手不能持械。负重分段惩罚、攻击冷却状态机；克制三角：锐克无甲、甲克锐、钝克甲。
- **近战这一下用哪把家伙，收敛成一个纯函数**：手里的东西能打人就用它打，打不了人的就用拳脚——**空手和持弓/持弩贴脸都算空手近战**（钝伤、低伤害、快冷却），枪走枪托 profile，近战武器原样。一条规则，没有散落各处的 if 分支。
- **丧尸穿生前的日常着装**：腐皮几乎不防护，丧尸的防御来自它死时身上那身衣服（随机着装池，穿衣率约 85%）；**精英丧尸穿护甲，是手写的具名预设、不进随机池**。

### 噪音 / 弹药 / 门 —— 让"强"付出代价的三个系统（§5 / §6）

这三个系统是**远程武器的代价来源**：枪不靠削数值变弱，而是靠"招怪 + 打不起"被系统压住。

- **噪音系统**：开火/近战/走路/开门/撬锁/砸门都发声，每种行为一个传播半径。**闲着的敌人听见会走过来查看**（复用既有"最后目击点"通道，不新造 AI 状态机）；已经在追人的敌人不会被声音拽走注意力。声音**不吃墙体遮挡**（会绕会穿），与吃遮挡的视线不同。
  - 梯度锚在两个既有常量上，不是拍脑袋：**弓弩 ≤ 丧尸嗅觉半径**（放箭不额外招怪）、**枪 ≫ 丧尸夜间视距**（一枪把屏幕外看不见你的东西全拽过来）。走路 < 弓弩 < 近战 < 砸门 ≪ 枪。
  - **听者语义分两类**：**战斗声不分敌我**（有人在打架，谁听见都过来——被围就真的滚雪球）；**只有脚步声分敌我**（否则丧尸会被彼此的脚步吸引、挤成一坨）。玩家操控的单位**永不被任何噪音牵着走**（硬安全阀，有回归单测钉死）。
- **弹药系统**：短/中/长子弹 + 鹿弹 + 箭；每次射击的耗弹量由连发数派生（多弹丸不相乘，一发壳就是一发）。子弹靠**子弹零件**复装（1 个零件按口径产 8/5/4/2 发），而零件以搜刮为主、刻意稀缺 —— 枪**强，但打不起**。**没弹药的枪自动降级成枪托近战**（复用既有贴脸枪托，不新造状态）。
- **弓弩体系**：5 把弓 + 3 把弩 + 4 种箭。**箭反过来改写弓的属性**（伤害/穿透/射程/冷却/散布按箭种做组合修正），同一把弓搭不同箭是不同的武器。箭不吃火药、**射出去能捡回来**（基础 25%，读过《弓与箭之道》50%），代价是弓没有枪托——贴脸只能挨打。弓＝弱但养得起，枪＝强但打不起。
- **门系统**：门有**开/关/锁**三态。**关着 = 挡人 + 挡视线 + 断寻路**（同一个墙层碰撞体 + 同一条导航洞承载，没有第二套判定）。**丧尸不会开门，只会砸**（门本身是可破坏结构，砸门整条链路复用既有破防系统）；**劫掠者会推门进来**。锁着的门可**撬**（安静、慢、要铁丝、失败断丝）或**砸**（快、全场皆知）——**砸永远更快，撬永远更静**，又一个噪音 vs 效率的取舍。

### 医疗与伤残（§5）

- **伤残能力系统**：部位损毁/切除影响对应能力（移动/操作/视觉…），断肢连带禁装与失手。
- **医疗与手术**：微伤自愈、手术即刻恢复/隔日重做等治疗规则。
- **卧床养病**：休养质量分三档——**在勤**（站岗/生产/读书/出门，该恶化就恶化）、**打地铺**（在养病但没床）、**睡床**（养病且占着一张床，加成吃满）。**床是一人一张**的实体家具，不够就得再造；躺下养病的人**顶掉自己那一班的产出**，于是"让他歇一天"永远是一笔要算的账。
- **感染是两条进度条的竞速**：伤口的**恶化条**满了就死人，**治疗条**先满则清除感染——用药只能加快治疗条，**不能减慢恶化条**。药分三档（药效 × 副作用各不相同），指派疗程后按人自动扣药、断药即清疗程。截肢是可选的保命手段。

### 装备 / 持械 / 制作（§6 / §7）

- **11 槽穿戴系统**：占用/校验/断肢禁装/替换/卸装，聚合"哪些部位被护甲覆盖"供战斗层只读消费；左右手主手/副手各持一把推导 GripMode。**成对装备（手套/鞋）一件占一槽——戴一只只护一只手，两件才护全**；断掉一只手只卸那一侧。
- **材料目录**：金属/木石/纺织/化工/医疗/弹药等类目，字符串键、可堆叠、搜刮与制作共用同一条链路。
- **配方 + 工作台三工具槽**：配方 = 材料 + 需要工具 + 制作者门槛（**读过的书**）→ 产物；`CraftingLogic` 出纯函数"能不能做/缺什么/扣什么产什么"，调用方去库存实扣实产。**没有通用技能系统**——角色能力只由「手写的专属效果」和「读过的书」承载。
- **角色专属效果（authored）**：具名角色带三级专属效果，升级条件各不相同；其中一位的等级**随营地人数涨落、会倒退**。全项目**百分比加成一律乘算**（加算会让没有手的人凭空获得 3% 操作能力）。
- **生产工时化**：配方带 `WorkHours`，在工作台上按夜间时间推进、可中断续做（配合双班制）。
- **武器改装**：按武器部位挂改装件，同部位不叠、异部位可叠；改装要在**改装台**上做（工作台的二级产物，得先搜到工具、读完书才造得出——"改枪"天然是中期解锁）。其中三种改装**重定义枪托近战的打法**：**利爪型**（绑刃挥砍→锐器切割）/ **创伤型**（枪托改铁锤→钝伤加重，代价是持握与射击精度变差）/ **刺刀型**（枪口挂刺刀→刺击穿透，最快最安静）。一把枪只能选其中一种——**这是在给"被贴脸之后怎么办"下注**。
- **读书活动**：夜间指派谁读哪本书、寻路到座位、读满解锁配方或被动加成（如箭矢回收率翻倍）；座位家具梯度（板凳→木椅→沙发预留）。

### 营地 / 防御 / 阵营（§4 / §5）

- **平地四面围栏 + 南北门**：围栏/大门有可破坏 HP、建造等级（等级→血量）；袭营破防择结构按边缘距离砸墙/贴近。
- **守卫岗位**：哨塔（+射程/视野）/ 屋顶平台 / 暗哨（先手一击）。
- **夜防对抗 + 双班制**：站岗者以视力/听力/岗哨加成合成**警戒力**，袭击者以光照/服饰/距离/遮蔽合成**潜行力**，双方 roll 对抗；发现后玩家选**三档响应**（低危仅守卫/中危唤醒生产者/高危全员），叫醒者次相位吃疲劳 debuff；未发现→敌人先手或静默偷窃。
- **三方阵营 + 敌对矩阵**：幸存者/丧尸/劫掠者；`Factions.IsHostile` 统一裁定敌我，运行时可切阵营/换择敌池。三阵营脚下描边（己方白/中立蓝/敌对红）。
- **围栏升级与拆除回收**：围栏可按边下令逐格加固/修补；家具与工事可拆，**返还向下取整**（木结构还一半木料、一半废木料，废木料要熬胶才粘得回木料）——所以拆了再建永远是亏的，"建错地方"是有代价的。
- **家具放置校验**：实心家具（工作台/改装台/烹饪台/床…）**不许贴着门和围栏放**——否则玩家能用家具把敌人的进攻路线堵死，或者把自己的门封上。家具**人能跨过去，但会减速**：它挡的是效率不是通路。（沙袋是刻意的例外：它非实心、不挖导航洞，本来就是给人翻的掩体。）
- **存档**：一天两次自动检查点（两顿聚餐），轮转保留最近若干个；版本不符即拒读、不做迁移。玩家没有手动存档——**留一串历史档是设计前提，不是补丁**。

### 光照视野 / 时限 / 同伴系统（§5）

- **光照与视野系统**：受光照与障碍制约的**锥形视野**——光线越差视距/视角越差。视野外走**遮暗 + 检测**双层（变暗 + 敌人/可交互物不揭示）；**丧尸/袭击者同吃**锥形视野+遮挡+光照，**潜行由此自然涌现**（绕背/走暗处/不持光源=不被发现），无需单独潜行系统。丧尸另有短程全向**嗅觉兜底**。手持光源（手电/火把）。适用探索关全程 + 营地夜间（营地白天全可见豁免）。
- **世界级时限系统**：隐性推进的全局倒计时框架 + 一处剧情调查点解锁 HUD 倒计时显示的机制（`HordeTimeline` / `LookoutSighting`；触发条件与叙事见设计文档）。
- **同伴系统（含犬类 actor）**：可招募角色带专属效果；其一为可站岗、可战斗、可被杀的犬类同伴——高闪避/高移速/低伤害的缠斗单元，有独立的进食与装备（狗装备制作/穿戴）逻辑，专属效果随共同存活时间升级。

### 经济 / 物品 / 库存 / 搜刮 / 剧情框架（§7 / §8）

- **货币（白银）与商人交易**：卖出/买入价率、商人来访日程与接替链的纯逻辑（`MerchantTrade` / `MerchantSchedule` / `MerchantLineage`）。
- **营地共享库存**：食物/书/护甲/武器/材料五类物品增删查（不做每人背包）。
- **烹饪**：每种食材有一个**热量点**，在**烹饪台**上凑够一份饭的热量就能做出食物；烹饪台两个槽位（锅/烤架），装上能让每份饭更省。份数 = 总热量 ÷ 每份需求，向下取整——**凑不满一份就做不成，多出来的零头浪费掉**。
  ⚠️ **这些数字游戏里一个都不显示**：几只老鼠够一顿饭、口粮值不值得下锅，是**玩家自己试出来的**。凑不够时按钮只是不可用、不解释原因；零头浪费不提示。这是刻意的——把答案印在 UI 上，这套机制就没有了。（数值本身在数值表 wiki 里，那是给设计者调的，见下。）
- **搜刮/探索**：探索关踏入搜刮点整批掉落 + 环境叙事（一次性、跨关持久去重，`ExplorationCache`）；容器 loot、剧情发现点日记（lore 物品带回营地阅读）。
- **剧情框架**：`StoryFlags` 字符串态状态机（引擎无剧情整数字段），按条件播放**手写剧情**——教学关、收留·放逐·处决抉择面板等，均为按条件触发 authored 内容的机制框架（内容本身见设计文档）。
- **昼夜循环 / 饥饿（多级阶梯）/ 聚餐气泡 / 游戏结束判定**（全体幸存者死亡→暂停 + 模态）。

**分层原则**：`DeadSignal.Combat` 只出纯函数规则（引擎，零依赖；含部位/护甲/弹道/多弹丸/弹药/弓弩/丧尸着装）。营地/装备/制作/守卫等系统的**纯逻辑**放在 `godot/scripts` 里不引 Godot 类型的文件（`ApparelSlots` / `WeaponLoadout` / `Recipe` / `CraftingLogic` / `CraftingJob` / `WeaponModLogic` / `Factions` / `BreachLogic` / `CampStructure` / `NoiseLogic` / `DoorLogic` / `AmmoSource` / `HordeTimeline` / `VisionLogic` / `NightWatchContest` / `ShiftSchedule` / `CookingLogic` / `BedrestLogic` / `PlacementRules` / `SalvageLogic` / `SaveCodec` / `SurvivorPerks` / `MerchantLineage` / `GameOverCondition` …），以 **Link 方式**编进 `DeadSignal.Combat.Tests` 单测覆盖（先红后绿）；空间执行（寻路/碰撞/弹道飞行/伤害施加/噪音广播/开关门重烘焙导航/遮暗渲染）才写进 Godot 运行时层。随机一律走可注入的 `IRandomSource`（测试用 `SequenceRandomSource` 复现）。

**关于 Sim 的数字**：模拟器跑的是 **1v1、无距离、无走位、无多目标**的引擎级对决，**不建模噪音与弹药**。所以霰弹枪的短射程代价、散射清群的优势、枪的"招怪 + 打不起"，Sim 一个都测不出——枪械胜率只能读作「无限弹药、贴脸、单挑」下的杀伤力天花板。空间侧机制只能实机校准。

## 构建 / 测试 / 运行

.NET 8 SDK 装在用户目录、**不在 PATH**，一律用 `~/.dotnet/dotnet` 并设 `DOTNET_ROOT`。Godot 用 4.7 .NET 版。

```sh
export DOTNET_ROOT=$HOME/.dotnet

# 构建（根 sln = 引擎 + 模拟器 + 测试，不含 godot 工程）
~/.dotnet/dotnet build DeadSignal.sln -c Release
~/.dotnet/dotnet build godot/DeadSignal.Godot.csproj   # godot 工程单独构建

# 测试（全绿是硬门禁；改战斗规则必须先红后绿）
~/.dotnet/dotnet test

# 模拟器（无参 = 聚合蒙特卡洛数值表；duel [路径] = 逐回合对决战报）
~/.dotnet/dotnet run --project src/DeadSignal.Sim

# 跑游戏（headless 跑 C# 需另 export PATH="$HOME/.dotnet:$PATH"）
/Applications/Godot_mono.app/Contents/MacOS/Godot --path godot      # 加 -e 进编辑器
```

## 数值表：本地 wiki

所有物品的数值（武器/护甲/材料/配方/弹药/医疗/光源/书籍/改装/家具/食物/角色…）都能在一个**跑在本机的网页**上看和改。
纯静态页面 + 一个 python 小服务，**零第三方依赖、离线可用、不联网、不碰数据库**。带分区、筛选、排序、行内编辑、新增/删除。

```sh
~/.dotnet/dotnet run --project tools/WikiExtract   # 从 C# 代码把数值抽成 JSON
python3 tools/wiki-serve.py                        # 起服务，打开它打印的地址
```

**数据流是单向的，这一点很重要：**

```
   C# 代码  ──────► tools/WikiExtract ──────►  docs/wiki/data/*.json  ──────►  网页
（唯一事实源）        （反射抽取，可重跑）          （设计表，在这里改）        （看 + 改）
      ▲                                                  │
      │                                                  │ 点「保存」写回文件
      └──────────  由人/agent 回写代码 + 跑测试门禁  ◄────┘
```

- **C# 始终是唯一事实源，游戏运行时不读这些 JSON。** 这是刻意的：数值改动必须过测试门禁和 Sim 复验，不能让一个手滑的格子直接进游戏。
- 抽取器**走反射遍历 catalog**——往 `WeaponTable` 里加一把枪、`Materials` 里加一种料，重跑就自动出现，不用改抽取器。
- 重跑**不会弄丢表里的改动**（"表赢代码"）：改过的值保留，新增/删除的行带着「待同步进代码」的标记活着，跑完打印一份**带代码位置的待同步清单**。
- 老的 `docs/*.xlsx` **已退役**（文件保留、不再是任何东西的数据源）。

细节见 `docs/wiki/README.md`。
