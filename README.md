# Dead Signal（死信号）

俯视角 2D 丧尸生存经营游戏 · 中短流程剧情向 · 对标《这是我的战争》 · Godot 4.7 + C# · 像素风

个人项目，不考虑发行。

## 概念

丧尸**白天休眠、夜晚游荡**。你经营一处庇护所：白天派 ≤3 人小队去手工设计的地点搜刮（人类劫掠者只在白天活动——主力出门时老家可能被抄）；夜晚实时度过，排班站岗/生产/读书，丧尸袭营时打实时 + 战术暂停的防御战。全员永久死亡、无主角；每个幸存者都有手写的招募剧情与支线，还有一条推动玩家行动的全局压力驱动主线。

> **剧情/世界观/结局细节一律不写进本文档**，以免剧透。完整叙事与玩法规则见设计文档（**含剧透**）：`docs/superpowers/specs/2026-07-04-dead-signal-design.md`。README 只讲**工程与玩法机制框架**。

## 核心循环

- **白天（丧尸休眠）**：从世界地图选点，派 ≤3 人小队进手工关卡搜刮物资、遭遇剧情、招募幸存者；天黑前必须回营。**人类劫掠者只在白天活动**——带谁出门、留谁守家是每日核心权衡。
- **夜晚（丧尸游荡）**：实时度过（1x/3x/8x 变速，事件自动拉回 1x），排班站岗/生产/修理/读书；丧尸袭营时进实时+战术暂停的防御战。
- **硬性双班作息**：昼班（探险）与夜班（站岗/生产）分开，非本班时段强制睡、不可操作，人手永远紧缺。
- **全局时间压力**：存在一条推动玩家推进主线、不能无限拖延的世界级压力（具体形态见设计文档）。

## 已实现系统

> 数值均为「拟定待调」占位，规则形态才是设计重点。以下只列机制；机制的剧情后果见设计文档。

### 战斗引擎（完成度最高的部分，纯 C# 零依赖 · 设计文档 §5）

- **部位树 + 独立 HP**：细到左右眼/鼻/下巴；头/颈/躯干归零致死，四肢致残，切除有连带（断臂失手）。
- **逐层护甲三段判定**：连续小数 roll，全伤/半伤转钝/挡下；护甲按部位覆盖（哪片甲护哪些部位）。
- **效果系统**：流血（扣储血量→轻/中/重度昏迷→出血致死）、切除、震荡（隔甲生效）、骨折（天然钝器专属）、0HP 部位钝伤磨损→永久损毁；**命中即使未破防也施加短暂减速**（RimWorld stagger 式全角色通用轻量效果——只降移速、不打断攻击冷却，与震荡严格区分）。
- **命中模型**：无命中率概念——近战必中，远程按误差角锥形弹道 + 射程/距离衰减；贴脸枪托近战；枪械冷却/三连发节奏。
- **持握 GripMode**：单手 / 单手改双手握（+15% 攻速）/ 双持两把单手（×0.70 攻速、误差角 ×1.25）/ 双手武器占两手；断手→该手不能持械。负重分段惩罚、攻击冷却状态机；克制三角：锐克无甲、甲克锐、钝克甲。

### 医疗与伤残（§5）

- **伤残能力系统**：部位损毁/切除影响对应能力（移动/操作/视觉…），断肢连带禁装与失手。
- **医疗与手术**：感染链、微伤自愈、手术即刻恢复/隔日重做、睡床加算等治疗规则。

### 装备 / 持械 / 制作（§6 / §7）

- **11 槽穿戴系统**：占用/校验/断肢禁装/替换/卸装，聚合"哪些部位被护甲覆盖"供战斗层只读消费；左右手主手/副手各持一把推导 GripMode。
- **材料目录 + 幸存者技能 SkillSet**（纺织/机械/化学/医疗/烹饪/战斗，经验累积升级）。
- **配方 + 工作台三工具槽**：配方 = 材料 + 需要工具 + 制作者门槛（读过的书/技能）→ 产物；`CraftingLogic` 出纯函数"能不能做/缺什么/扣什么产什么"，调用方去库存实扣实产。
- **生产工时化**：配方带 `WorkHours`，在工作台上按夜间时间推进、可中断续做（配合双班制）。
- **武器改装 WeaponMod**：按武器部位挂改装件，同部位不叠、异部位可叠。
- **读书活动**：夜间指派谁读哪本书、寻路到座位、读满解锁配方/技能；座位家具梯度（板凳→木椅→沙发预留）。

### 营地 / 防御 / 阵营（§4 / §5）

- **平地四面围栏 + 南北门**：围栏/大门有可破坏 HP、建造等级（等级→血量）；袭营破防择结构按边缘距离砸墙/贴近。
- **守卫岗位**：哨塔（+射程/视野）/ 屋顶平台 / 暗哨（先手一击）。
- **夜防对抗 + 双班制**：站岗者以视力/听力/岗哨加成合成**警戒力**，袭击者以光照/服饰/距离/遮蔽合成**潜行力**，双方 roll 对抗；发现后玩家选**三档响应**（低危仅守卫/中危唤醒生产者/高危全员），叫醒者次相位吃疲劳 debuff；未发现→敌人先手或静默偷窃。
- **三方阵营 + 敌对矩阵**：幸存者/丧尸/劫掠者；`Factions.IsHostile` 统一裁定敌我，运行时可切阵营/换择敌池。三阵营脚下描边（己方白/中立蓝/敌对红）。

### 光照视野 / 时限 / 同伴系统（§5）

- **光照与视野系统**：受光照与障碍制约的**锥形视野**——光线越差视距/视角越差。视野外走**遮暗 + 检测**双层（变暗 + 敌人/可交互物不揭示）；**丧尸/袭击者同吃**锥形视野+遮挡+光照，**潜行由此自然涌现**（绕背/走暗处/不持光源=不被发现），无需单独潜行系统。丧尸另有短程全向**嗅觉兜底**。手持光源（手电/火把）。适用探索关全程 + 营地夜间（营地白天全可见豁免）。
- **世界级时限系统**：隐性推进的全局倒计时框架 + 一处剧情调查点解锁 HUD 倒计时显示的机制（`HordeTimeline` / `LookoutSighting`；触发条件与叙事见设计文档）。
- **同伴系统（含犬类 actor）**：可招募角色带专属效果；其一为可站岗、可战斗、可被杀的犬类同伴——高闪避/高移速/低伤害的缠斗单元，有独立的进食与装备（狗装备制作/穿戴）逻辑，专属效果随共同存活时间升级。

### 经济 / 物品 / 库存 / 搜刮 / 剧情框架（§7 / §8）

- **货币（白银）与商人交易**：卖出/买入价率、商人来访日程与接替链的纯逻辑（`MerchantTrade` / `MerchantSchedule` / `MerchantLineage`）。
- **营地共享库存**：食物/书/护甲/武器/材料五类物品增删查（不做每人背包）。
- **搜刮/探索**：探索关踏入搜刮点整批掉落 + 环境叙事（一次性、跨关持久去重，`ExplorationCache`）；容器 loot、剧情发现点日记（lore 物品带回营地阅读）。
- **剧情框架**：`StoryFlags` 字符串态状态机（引擎无剧情整数字段），按条件播放**手写剧情**——教学关、收留·放逐·处决抉择面板等，均为按条件触发 authored 内容的机制框架（内容本身见设计文档）。
- **昼夜循环 / 饥饿（多级阶梯）/ 聚餐气泡 / 游戏结束判定**（全体幸存者死亡→暂停 + 模态）。

## 项目结构

```
src/DeadSignal.Combat/           战斗规则引擎（纯 C# 类库，零依赖；空间问题不进这里）
src/DeadSignal.Sim/              蒙特卡洛模拟器 + 逐回合对决战报（拉数值表校准方向）
tests/DeadSignal.Combat.Tests/   xUnit 单测（69 个文件、约 905 个 [Fact]/[Theory]，
                                 Theory 展开后运行用例约 980；引擎 + Link 进来的消费层纯逻辑）
godot/                           Godot 4.7 工程 —— RimWorld 式操控的可玩场景
  scripts/                       消费层：营地/战斗表现/UI/装备/制作/守卫/剧情 编排 + 纯逻辑
  data/                          数据驱动配置（daynight/camp/meal_bubbles…）
docs/superpowers/specs/          设计文档（单一事实源，含剧透）
docs/research/                   数值基线报告 / 对决战报 / 美术可行性调研
docs/TODO.md                     待办清单（完成一项删一项）
```

**分层原则**：`DeadSignal.Combat` 只出纯函数规则（引擎，零依赖）。营地/装备/制作/守卫等系统的**纯逻辑**放在 `godot/scripts` 里不引 Godot 类型的文件（`ApparelSlots` / `WeaponLoadout` / `SkillSet` / `Recipe` / `CraftingLogic` / `CraftingJob` / `Factions` / `BreachLogic` / `CampStructure` / `HordeTimeline` / `LookoutSighting` / `VisionLogic` / `NightWatchContest` / `ShiftSchedule` / `DougBruceBond` / `DogApparel` / `MerchantLineage` / `HeldLightState` / `GameOverCondition` …），以 **Link 方式**编进 `DeadSignal.Combat.Tests` 单测覆盖（先红后绿）；空间执行（寻路/碰撞/弹道飞行/伤害施加/遮暗渲染）才写进 Godot 运行时层。随机一律走可注入的 `IRandomSource`（测试用 `SequenceRandomSource` 复现）。

## 构建 / 测试 / 运行

.NET 8 SDK 装在用户目录、**不在 PATH**，一律用 `~/.dotnet/dotnet` 并设 `DOTNET_ROOT`。Godot 用 4.7 .NET 版。

```sh
export DOTNET_ROOT=$HOME/.dotnet

# 构建（根 sln = 引擎 + 模拟器 + 测试，不含 godot 工程）
~/.dotnet/dotnet build DeadSignal.sln -c Release
~/.dotnet/dotnet build godot/DeadSignal.Godot.csproj   # godot 工程单独构建

# 测试（全绿是硬门禁；改战斗规则必须先红后绿）
~/.dotnet/dotnet test

# 模拟器（无参 = 聚合蒙特卡洛数值表；duel [路径] = 逐回合对决战报）
~/.dotnet/dotnet run --project src/DeadSignal.Sim

# 跑游戏（headless 跑 C# 需另 export PATH="$HOME/.dotnet:$PATH"）
/Applications/Godot_mono.app/Contents/MacOS/Godot --path godot      # 加 -e 进编辑器
```

试玩操控：左键选人 / 右键指令 / 空格暂停 / 1·2·3 变速 / T 跳昼夜。
